- 关于 Lua 脚本特效 -

        YukkuriMovieMaker4 的"Lua 脚本"视频特效使用
        Lua 语言编写脚本。
        脚本引擎采用 MoonSharp（兼容 Lua 5.2）。

        * 本特效为独立实现，与 AviUtl2 的脚本规范不同。
        * 与 AviUtl2 的脚本文件（*.anm2 等）为不同规范。
        * 由于 IsAviUtlSupported = false，不支持面向 AviUtl 的 EXO 输出。


- 注意事项 -

        ○ 脚本执行引擎为 MoonSharp（兼容 Lua 5.2）。
        ○ 可用的标准库仅限以下内容：
            table, string, math, bit32
            * io、os、debug、ffi、require 等不可使用。
        ○ 脚本超时时间为 5000ms（5 秒）。
            超时时，执行将被强制终止并视为错误。
        ○ 脚本无法访问文件系统（包括 require）。
        ○ 不支持从脚本内部调用外部脚本。
        ○ 像素操作（getpixel/setpixel/getpixeldata）会访问实际输入图像，
            因此仅在首次访问时发生 GPU→CPU 的数据传输。
            不需要时省略调用可加快处理速度。


- 执行引擎与自动加速 -

        包含像素处理（getpixel/setpixel 双重循环）的脚本会被解析语法，
        并自动分配到更快的引擎。
        坐标、旋转、不透明度等普通脚本仍照旧由 MoonSharp 执行，行为不变。

        ○ 自动分配（未指定指令时）
                · 自像素完结的像素映射（见下方条件）→ CPU 内核（高速、精确）
                · 上述以外的像素操作 → 原生（LuaJIT，仅在已部署时）
                · 无像素操作 → MoonSharp

        ○ 引擎指定指令（脚本首行的注释）
                --!cpu       ：CPU 内核。运行时编译为表达式树，
                               在所有 CPU 核心上并行执行（与 MoonSharp 完全一致）。
                --!gpu       ：GPU 内核。作为 Direct2D 像素着色器执行，
                               消除 GPU→CPU 传输，速度最快。
                               因 float32 精度，结果可能有极小差异（需主动选用）。
                --!native    ：LuaJIT（独立进程）。高速且与 MoonSharp 一致。
                               需要部署 native 文件夹。
                --!moonsharp ：强制使用传统的 MoonSharp。
                ※ 指令不可用/不支持时，会自动回退到较低的引擎
                  （输出不会损坏）。

        ○ 可成为高速内核的语法（纯像素映射）
                只有以下形式的双重循环才会被内核化。
                    for y = 0, obj.h - 1 do
                        for x = 0, obj.w - 1 do
                            local r, g, b, a = obj.getpixel(x, y)
                            obj.setpixel(x, y, r2, g2, b2 [, a2])
                        end
                    end
                · 主体中仅可使用四则运算、math.*、比较、阈值写法
                  “条件 and 值1 or 值2”以及 local 变量。
                · 仅可引用正在处理的自像素（getpixel(x, y)）
                  以及 obj/scene/time 等值。
                · 含有邻域参照（模糊等）、写入其他坐标、getpixeldata、
                  getobject、表/字符串操作、if 语句等的脚本不会被内核化，
                  会自动由传统引擎执行。

        示例（让白色图形变红）：
                --!gpu
                for y = 0, obj.h - 1 do
                    for x = 0, obj.w - 1 do
                        local r, g, b, a = obj.getpixel(x, y)
                        obj.setpixel(x, y, r, g / 2, b / 2, a)
                    end
                end


- 参数 -

        特效面板包含以下 4 个轨道条（支持动画）。
        各值在脚本中可通过 obj.track0 ～ obj.track3 引用。

        Track0：参数 0（默认值：0，范围：最小值 ～ 最大值）
        Track1：参数 1（默认值：0，范围：最小值 ～ 最大值）
        Track2：参数 2（默认值：0，范围：最小值 ～ 最大值）
        Track3：参数 3（默认值：0，范围：最小值 ～ 最大值）

        * 每个轨道条均支持动画滑块（含插值）。
        * 可通过脚本编辑器工具栏导入/导出 .lua 文件。


- 全局变量 -

        以下变量可在脚本的全局作用域中直接引用。

        time          : 相对于对象的当前时间（秒）[只读]
                        * 与 obj.time 的值相同。
        frame         : 相对于对象的当前帧编号 [只读]
                        * 与 obj.frame 的值相同。
        totalframe    : 对象的总帧数 [只读]
                        * 与 obj.totalframe 的值相同。
        framerate     : 帧率（fps）[只读]
                        * 与 obj.framerate 的值相同。
        timelineframe : 相对于整个时间轴的当前帧编号 [只读]
                        * obj.frame 是从对象起点的相对帧，
                          而此值是从时间轴起点的绝对帧。
        timelinetime  : 相对于整个时间轴的当前时间（秒）[只读]
        layer         : 对象所在的图层编号 [只读]
                        * 与 obj.layer 的值相同。
        obj           : 对象信息表（参见下文"obj 表"）
        scene         : 场景信息表（参见下文"scene 表"）
        ymm4          : YMM4专有的元数据表 (参见下文“ymm4 表”)
        anim          : 动画专用工具表 (参见下文“anim 表”)


- scene 表 -

        scene.width   : 场景（项目）的水平分辨率（像素）[只读]
        scene.height  : 场景（项目）的垂直分辨率（像素）[只读]


- obj 表 - 变量 -

        以下变量保存对象当前的绘制信息。
        * 未标注"只读"的变量可在脚本中修改，修改后的值将反映到绘制中。

        ○ 坐标·偏移
        obj.x         : 显示基准坐标 X（可修改）
                        * 对应 AviUtl2 规范的"显示基准坐标 X"。
                        * 内部由 DrawDescription.Draw.X 设置，
                          脚本执行后更改内容才会生效。
        obj.y         : 显示基准坐标 Y（可修改）
                        * 内部由 DrawDescription.Draw.Y 设置。
        obj.z         : 显示基准坐标 Z（可修改）
                        * 内部由 DrawDescription.Draw.Z 设置。
        obj.ox        : 中心偏移 X（可修改）
                        * 内部由 DrawDescription.CenterPoint.X (inDesc) 设置。
                        * 对应 AviUtl2 规范的"从基准坐标的相对坐标 X (obj.ox)"，
                          但在本插件中作为 CenterPoint（中心点偏移）处理。
        obj.oy        : 中心偏移 Y（可修改）
                        * 内部由 DrawDescription.CenterPoint.Y (inDesc) 设置。

        ○ 缩放·变形
        obj.zoom      : 缩放比例（可修改）
                        * 等倍为 1.0 而非 0.0（兼容 AviUtl2）。
                        * 内部由 ZoomX 与 ZoomY 的平均值（zoomAvg）计算得出。
                        * 输出时以 obj.zoom * (1 ± obj.aspect) 展开为 X/Y 各自的 Zoom。
        obj.aspect    : 宽高比（可修改），范围：-1.0 ～ 1.0
                        * 正值在 X 方向缩小，负值在 Y 方向缩小。
                        * 内部由 (ZoomX - ZoomY) / (ZoomX + ZoomY) 计算得出。
        obj.sx        : X方向缩放率 (可修改) 1.0为100%
                        ※ YMM4专有的精确独立缩放控制。
                        ※ 优先于 zoom/aspect 的修改生效。
        obj.sy        : Y方向缩放率 (可修改) 1.0为100%
                        ※ YMM4专有的精确独立缩放控制。
                        ※ 优先于 zoom/aspect 的修改生效。
        obj.alpha     : 不透明度（可修改），范围：0.0 ～ 255.0
                        * AviUtl2 为 0.0～1.0，本插件为 0.0～255.0。
                        * 内部由 DrawDescription.Opacity * 255 设置。
                        * 输出时除以 255 并归一化为 0.0～1.0（含 Clamp）。
                        * 默认脚本设置了 obj.alpha = math.min(time * 255, 255)，
                          作为淡入的默认实现。

        ○ 旋转
        obj.rx        : X 轴旋转角度（可修改），单位：度（°），360.0 为一圈
                        * 内部由 DrawDescription.Rotation.X 设置。
        obj.ry        : Y 轴旋转角度（可修改），单位：度（°），360.0 为一圈
                        * 内部由 DrawDescription.Rotation.Y 设置。
        obj.rz        : Z 轴旋转角度（可修改），单位：度（°），360.0 为一圈
                        * 内部由 DrawDescription.Rotation.Z 设置。
        obj.rxr       : X轴旋转角度(弧度) (可修改)
                        ※ 若与rx同时修改，将优先反映此值。
        obj.ryr       : Y轴旋转角度(弧度) (可修改)
                        ※ 若与ry同时修改，将优先反映此值。
        obj.rzr       : Z轴旋转角度(弧度) (可修改)
                        ※ 若与rz同时修改，将优先反映此值。

        ○ 图像尺寸
        obj.w         : 输入图像的水平像素数 [只读]
                        * 内部为从 D2D 的 GetImageLocalBounds 计算得出的整数值。
        obj.h         : 输入图像的垂直像素数 [只读]
                        * 内部为从 D2D 的 GetImageLocalBounds 计算得出的整数值。
        obj.hw        : 输入图像水平像素数的一半 (= obj.w / 2) [ReadOnly]
        obj.hh        : 输入图像垂直像素数的一半 (= obj.h / 2) [ReadOnly]
        obj.diagonal  : 输入图像的对角线长度 (= math.sqrt(w*w + h*h)) [ReadOnly]
        obj.cx        : 图像中心 X 坐标（= obj.w / 2）[只读]
                        * 像素操作中中心坐标的参考值。
        obj.cy        : 图像中心 Y 坐标（= obj.h / 2）[只读]

        ○ 时间·帧
        obj.time      : 相对于对象的当前时间（秒）[只读]
                        * 与全局变量 time 的值相同。
                        * 内部由 EffectDescription.ItemPosition.Time.TotalSeconds 设置。
        obj.frame     : 相对于对象的当前帧编号 [只读]
                        * 与全局变量 frame 的值相同。
                        * 内部由 EffectDescription.ItemPosition.Frame 设置。
        obj.totalframe: 对象的总帧数 [只读]
                        * 与全局变量 totalframe 的值相同。
                        * 内部由 EffectDescription.ItemDuration.Frame 设置。
        obj.framerate : 帧率（fps）[只读]
                        * 与全局变量 framerate 的值相同。
                        * 内部由 EffectDescription.FPS 设置。

        ○ 轨道条
        obj.track0    : 特效面板 Track0 的当前值 [只读]
                        * 动画插值后的值。
        obj.track1    : 特效面板 Track1 的当前值 [只读]
        obj.track2    : 特效面板 Track2 的当前值 [只读]
        obj.track3    : 特效面板 Track3 的当前值 [只读]

        ○ 图层
        obj.layer     : 对象所在的图层编号 [只读]
                        * 与全局变量 layer 的值相同。
                        * 内部由 EffectDescription.Layer 设置。


- obj 表 - 变量修改与绘制反映 -

        脚本执行后，仅以下变量会被回读到绘制信息中（生效）。

        生效的变量：
            obj.x, obj.y, obj.z
            obj.ox, obj.oy
            obj.zoom, obj.aspect
            obj.alpha
            obj.rx, obj.ry, obj.rz

        * obj.w、obj.h、obj.cx、obj.cy、obj.frame、obj.time、
          obj.totalframe、obj.framerate、obj.track0 ～ obj.track3、
          obj.layer 为只读，修改后不会反映到绘制中。

        输出值计算公式：
            ZoomX = obj.zoom * (1.0 + obj.aspect)
            ZoomY = obj.zoom * (1.0 - obj.aspect)
            Opacity = Clamp(obj.alpha / 255.0, 0.0, 1.0)
            Draw = (obj.x, obj.y, obj.z)
            CenterPoint = (obj.ox, obj.oy)
            Rotation = (obj.rx, obj.ry, obj.rz)


- obj 表 - 函数 -

        ○ obj.getobject(tag [, frame])
                获取具有指定标签的其他对象的绘制信息。
                ※ 标签与目标项目“备注”中输入的字符串相匹配。
                ※ 指定第二个参数 frame 时，获取该时间轴帧的数值。省略时获取当前帧。
                ※ 当多个项目拥有相同标签时，优先返回在目标帧显示中的项目。
                  若没有显示中的项目，则返回第一个匹配的项目。
                ※ 获取的值是项目自身的坐标、旋转和缩放。
                  不包含组控制、摄像机控制以及上层特效造成的变换。
                ※ 各数值使用与 obj 表相同的单位体系。
                tag     : 设置在目标项目备注中的字符串
                frame   : 要获取的时间轴绝对帧 (可省略)
                返回值  : 对象信息表。当不存在匹配项目时返回 nil。
                          .exist : 是否在目标帧显示中 (boolean)
                          .x     : 显示基准坐标 X
                          .y     : 显示基准坐标 Y
                          .z     : 显示基准坐标 Z
                          .zoom  : 缩放率 (1.0 为原始大小)
                          .sx    : X 方向缩放率 (= zoom)
                          .sy    : Y 方向缩放率 (= zoom)
                          .rx    : X 轴旋转角度 (度, 恒为 0)
                          .ry    : Y 轴旋转角度 (度, 恒为 0)
                          .rz    : Z 轴旋转角度 (度)
                          .rxr   : X 轴旋转角度 (弧度, 恒为 0)
                          .ryr   : Y 轴旋转角度 (弧度, 恒为 0)
                          .rzr   : Z 轴旋转角度 (弧度)
                          .alpha : 不透明度 (0.0 ～ 255.0, 已应用淡入淡出)
                          .layer : 所在图层编号
                示例：
                    local funnel = obj.getobject("funnel")
                    if funnel then
                        obj.x = funnel.x
                        obj.y = funnel.y
                    end
                固定为发射时坐标的示例 (发射源备注为 "funnel")：
                    local fire = timelineframe - frame
                    local src = obj.getobject("funnel", fire)
                    if src then
                        obj.x = src.x
                        obj.y = src.y
                    end

        ○ obj.getpixel(x, y)
                获取指定坐标的像素信息。
                * 首次调用时发生 GPU→CPU 的图像数据传输（存在处理开销）。
                * 像素坐标以图像左上角为 (0, 0)，向右和向下为正方向。
                * 返回值为将预乘 Alpha 逆转换后的直通 Alpha 等效值。
                x      ：X 坐标（0 ～ obj.w - 1）
                y      ：Y 坐标（0 ～ obj.h - 1）
                返回值 ：r, g, b, a 共 4 个值（多值返回）
                          r = 红色分量（0.0 ～ 255.0）
                          g = 绿色分量（0.0 ～ 255.0）
                          b = 蓝色分量（0.0 ～ 255.0）
                          a = Alpha 值（0.0 ～ 255.0）
                          * 超出范围的坐标返回 (0, 0, 0, 0)。
                示例：r, g, b, a = obj.getpixel(10, 20)

        ○ obj.setpixel(x, y, r, g, b [, a])
                覆写指定坐标的像素信息。
                * 调用时像素数据自动标记为"已修改"，
                  脚本结束后输出修改后的图像。
                * 内部以预乘 Alpha 格式（BGRA）保存，
                  setpixel 时进行预乘转换。
                x      ：X 坐标（0 ～ obj.w - 1）
                y      ：Y 坐标（0 ～ obj.h - 1）
                r      ：红色分量（0.0 ～ 255.0）
                g      ：绿色分量（0.0 ～ 255.0）
                b      ：蓝色分量（0.0 ～ 255.0）
                a      ：Alpha 值（省略时为 255.0）
                示例：obj.setpixel(10, 20, 255, 0, 0, 128)

        ○ obj.getpixeldata()
                返回用于批量访问所有像素数据的代理对象。
                * 与 getpixel/setpixel 相比，像素操作速度更快。
                * 首次调用时发生 GPU→CPU 的图像数据传输（存在处理开销）。
                返回值 ：PixelDataProxy 对象
                          .width  ：图像水平像素数（= obj.w）
                          .height ：图像垂直像素数（= obj.h）
                          :get(index)        ：通过索引获取通道值
                          :set(index, value) ：通过索引设置通道值

                索引规则：
                    索引从 1 开始（遵循 Lua 惯例）。
                    通道顺序为 R、G、B、A 循环。
                    pixel(x, y) 的 R 通道索引 =
                        (y * obj.w + x) * 4 + 1
                    pixel(x, y) 的 G 通道索引 =
                        (y * obj.w + x) * 4 + 2
                    pixel(x, y) 的 B 通道索引 =
                        (y * obj.w + x) * 4 + 3
                    pixel(x, y) 的 A 通道索引 =
                        (y * obj.w + x) * 4 + 4

                    * 修改 Alpha（通道 4）时，RGB 也会按比例转换。
                    * 修改 RGB 通道时，会进行预乘转换。
                    * 获取值与 get(index) 相同，为直通 Alpha 等效值。

                示例：
                    local pd = obj.getpixeldata()
                    local idx = (10 * obj.w + 20) * 4 + 1  -- (x=20, y=10) 的 R 通道
                    local r = pd:get(idx)
                    pd:set(idx, 128)

        ○ obj.putpixeldata()
                为兼容性而定义的空实现函数（no-op）。
                * AviUtl 脚本中 getpixeldata/putpixeldata 组合使用，
                  但本插件中 setpixel/set 的修改会自动反映，
                  因此无需调用此函数。

        ○ obj.rand(st, ed [, seed [, frame]])
                返回指定范围 st ～ ed 内整数的确定性随机数。
                * seed 与 frame 相同时始终返回相同的值。
                * 省略 seed 时按 0 处理，省略 frame 时使用当前帧 (obj.frame)。
                * 在传统引擎 (MoonSharp) 与高速运行时 (LuaJIT) 上返回相同的值。
                  若需要不依赖引擎的小数随机数，请使用 anim.rand。
                st      : 范围下限（小数向下取整为整数）
                ed      : 范围上限（小数向下取整为整数）
                seed    : 随机种子（可省略，默认 0）
                frame   : 用于变化的帧（可省略，默认 obj.frame）
                返回值  : st ～ ed 的整数
                示例：obj.ox = obj.ox + obj.rand(-10, 10, 1, obj.frame)

        ○ obj.load(目标, ...)
                将对象缓冲区替换为生成的内容。支持 "figure"(图形) / "text"(文本) /
                "image"(图像文件) / "movie"(视频文件)。加载后 obj.w / obj.h 等会更新为
                生成尺寸，紧接着的 getpixel 也可读取。传统引擎与高速引擎均可运行。

                obj.load("figure", 名称, 颜色, 大小 [, 线宽 [, 纵横比]])
                        生成图形并替换缓冲区(居中)。
                        名称   : 円 / 四角形 / 三角形 / 五角形 / 六角形 / 星形
                                 (也可用英文 circle, rect, triangle, pentagon,
                                 hexagon, star)。
                        颜色   : 填充色 0xRRGGBB。
                        大小   : 基准大小(像素)。
                        线宽   : 轮廓宽度(可省略)。0 为填充。
                        纵横比 : 变形 -1.0 ～ 1.0(可省略)。
                        示例：obj.load("figure", "円", 0xff3366, 200)

                obj.load("text", 字符串)
                        以 obj.setfont 指定的字体、大小、颜色绘制字符串并替换缓冲区
                        (使用 DirectWrite 绘制)。
                        示例：obj.setfont("Yu Gothic UI", 48); obj.load("text", "字幕")

                obj.load("image", 路径)
                        加载图像文件并替换缓冲区。
                        ※ 为安全起见会校验文件头，仅加载真正的图像
                          (PNG / JPEG / GIF / BMP / TIFF / WebP)。
                        示例：obj.load("image", "C:\\img\\logo.png")

                obj.load("movie", 路径 [, 时间])
                        加载视频文件指定时间的帧并替换缓冲区
                        (使用 Media Foundation 解码)。
                        时间 : 秒。省略时为 obj.time。
                        ※ 为安全起见会校验文件头，仅加载真正的视频
                          (MP4 / MOV / MKV / WebM / AVI / FLV / ASF)。
                        示例：obj.load("movie", "C:\\mov\\clip.mp4", obj.time)

        ○ obj.setfont(字体 [, 大小 [, 样式 [, 颜色]]])
                设置 obj.load("text") 使用的字体。
                样式 : 位标志。1=粗体, 2=斜体, 3=粗斜体。
                颜色 : 0xRRGGBB。
                示例：obj.setfont("Meiryo", 64, 1, 0xffffff)

        ○ obj.draw([ox, oy, oz [, 缩放 [, 不透明度 [, 纵横比]]]])
                将当前对象图像合成到累积缓冲区并绘制。
                多次调用会叠加绘制(残像、粒子等)。
                ox, oy, oz : 相对中心的偏移(默认 0)。
                缩放       : 1.0 为原始大小(默认 1.0)。
                不透明度   : 0.0 ～ 1.0(默认 1.0)。
                纵横比     : -1.0 ～ 1.0(默认 0)。
                ※ 调用后将抑制标准绘制，输出合成结果。

        ○ obj.drawpoly(x0,y0,z0, x1,y1,z1, x2,y2,z2, x3,y3,z3
                        [, u0,v0, u1,v1, u2,v2, u3,v3 [, 不透明度]])
                将对象图像以纹理方式绘制到 4 个顶点指定的四边形。
                顶点顺序为左上→右上→右下→左下。省略 UV 时使用整幅图像。
                ※ 省略 UV 而仅提供第 13 个参数时，将其用作不透明度(12 个坐标 + 不透明度)。
                ※ 使用仿射变换映射(平行四边形精确，非平行四边形为近似)。

        ○ obj.copybuffer(目标, 源)
                在缓冲区之间复制图像。按标识符首字符判定。
                  "obj"      : 对象图像缓冲区
                  "tmp"      : 临时缓冲区
                  "cache:名称" : 命名缓存(跨帧保留)
                示例：obj.copybuffer("tmp", "obj")  -- 保存
                      obj.copybuffer("obj", "tmp")  -- 恢复
                ※ 由于 YMM4 的结构，无法获取 "frm"(帧缓冲区 / 背景)。

        ○ obj.getvalue(目标)
                获取 obj 的当前值(与 obj.x、obj.track0 等相同)。
                目标   : "x","y","z","ox","oy","rx","ry","rz","zoom","aspect",
                         "alpha","sx","sy","track0"～"track3","time","frame" 等。
                返回值 : 数值(未知目标返回 0)。

        ○ obj.setoption(名称, 值) / obj.getoption(名称)
                设置 / 获取选项。未支持的项目也会被接受，不会中断脚本
                (getoption 返回设置值，未设置为 0)。
                当前会反映到绘制的项目：
                  "antialias" : 0 为最近邻，其它为线性插值(obj.draw / obj.drawpoly)。

        ○ obj.pixeloption(名称, 值)
                将像素操作的选项作为状态保存。



- ymm4 表 -

        保存YMM4专有元数据的表。全部为 ReadOnly。

        ymm4.group_index        : 组控制中的当前索引 (从0开始)
        ymm4.group_count        : 组控制中的总项目数
        ymm4.group_ratio        : 组内进度比例 (group_index / group_count)
        ymm4.timeline_totalframe: 整个时间轴的总帧数
        ymm4.timeline_totaltime : 整个时间轴的总时间(秒)
        ymm4.time_ratio         : 当前播放位置的比例 (0.0 ～ 1.0)
        ymm4.is_saving          : 视频是否正在导出中 (仅在导出时为true)
        ymm4.is_playing         : 是否正在播放中 (仅在播放时为true)
        ymm4.is_paused          : 是否处于暂停中 (仅在暂停时为true)
        ymm4.scene_id           : 当前场景的唯一识别字符串 (Guid)


- anim 表 -

        便于动画计算的纯工具函数及常量表。

        anim.tau                : 圆周率的两倍 (math.pi * 2)
        anim.e                  : 欧拉数 (math.exp(1))
        anim.phi                : 黄金分割率 ((1 + math.sqrt(5)) / 2)
        anim.sqrt2              : 2的平方根 (math.sqrt(2))
        anim.lerp(a, b, t)      : 将a与b按t(0.0～1.0)进行线性插值
        anim.smoothstep(e0, e1, x): 在e0和e1之间进行平滑过渡插值
        anim.smootherstep(e0, e1, x): e0 和 e1 之间更平滑的埃尔米特插值 (二阶导数连续)
        anim.clamp(v, lo, hi)   : 将值v限制在lo～hi范围内
        anim.map(v, a1, b1, a2, b2): 将范围a1～b1内的值v映射至a2～b2范围
        anim.norm(v, lo, hi)    : 获取值v在lo～hi范围内的比例(0.0～1.0)
        anim.wrap(v, lo, hi)    : 使值v在lo～hi范围内循环
        anim.pingpong(t, len)   : 在0～len之间往复变化的值
        anim.sign(v)            : 获取值v的符号 (正=1, 负=-1, 零=0)
        anim.oscillate(t, lo, hi, freq): 以freq频率在lo与hi之间平滑往复(正弦波)
        anim.triangle(t, freq)  : 以freq频率在0.0～1.0之间往复的三角波
        anim.square(t, freq)    : 以freq频率在1.0与0.0之间切换的方波
        anim.duration(t, dur)   : 针对持续时间dur的当前比例(0.0～1.0, 含clamp)
        anim.delay(t, d)        : 经过时间d后开始计时的值 (小于0则为0)
        anim.ease_in(t)         : 二次加速缓动
        anim.ease_out(t)        : 二次减速缓动
        anim.ease_in_out(t)     : 二次缓入缓出 (先加速后减速)
        anim.elastic(t)         : 弹性弹簧缓动
        anim.back(t)            : 略微越界后返回的缓动
        anim.step(edge, x)      : 若x大于等于edge则返回1.0，否则返回0.0
        anim.fract(v)           : 仅获取值v的小数部分
        anim.bounce(t)          : 弹跳缓动
        anim.hsv_to_rgb(h, s, v): 将HSV (色相0～360, 饱和度0～1, 明度0～1) 转为 RGB
        anim.rgb_to_hsv(r, g, b): 将RGB (0～255) 转为 HSV
        anim.noise(x, y, z)     : 1维～3维的平滑噪声 (0.0～1.0)
        anim.rand(min, max, s)  : 基于种子 s 的稳定随机数
        anim.len(x, y)          : 获取向量 (x, y) 的长度 (范数)
        anim.dist(x1, y1, x2, y2): 获取两点 (x1, y1) 和 (x2, y2) 之间的距离
        anim.dot(x1, y1, x2, y2): 获取两个向量 (x1, y1) 和 (x2, y2) 的点积
        anim.normalize(x, y)    : 归一化向量 (x, y) 返回方向向量 (Tuple)
        anim.polar(r, a)        : 通过半径 r 和角度 a(度) 获取 (x, y) 坐标的元组
        anim.rotate(x, y, a)    : 获取将坐标 (x, y) 旋转 a(度) 后的坐标元组
        anim.bezier(t, p0, p1, p2, p3): 计算 t(0.0～1.0) 时的三次贝塞尔曲线值

- 脚本示例 -

        ○ 淡入（默认脚本）
                obj.alpha = math.min(time * 255, 255)

        ○ 淡出
                obj.alpha = math.max((obj.totalframe - obj.frame) / obj.framerate * 255, 0)

        ○ 用 Track0 控制不透明度
                obj.alpha = obj.track0

        ○ 随时间旋转
                obj.rz = time * 90  -- 每秒旋转 90 度

        ○ 反转像素红色分量
                for y = 0, obj.h - 1 do
                    for x = 0, obj.w - 1 do
                        local r, g, b, a = obj.getpixel(x, y)
                        obj.setpixel(x, y, 255 - r, g, b, a)
                    end
                end

        ○ 使用 getpixeldata 进行灰度转换（高速）
                local pd = obj.getpixeldata()
                local w = pd.width
                local h = pd.height
                for y = 0, h - 1 do
                    for x = 0, w - 1 do
                        local base = (y * w + x) * 4
                        local r = pd:get(base + 1)
                        local g = pd:get(base + 2)
                        local b = pd:get(base + 3)
                        local a = pd:get(base + 4)
                        local gray = r * 0.299 + g * 0.587 + b * 0.114
                        pd:set(base + 1, gray)
                        pd:set(base + 2, gray)
                        pd:set(base + 3, gray)
                    end
                end

        ○ 用 Track0 控制缩放（1.0 = 等倍）
                obj.zoom = obj.zoom * (1 + obj.track0 / 100)


- AviUtl 脚本的部分兼容 -

        提供兼容层，使 AviUtl 动画效果脚本(*.anm 系列)中涉及坐标、缩放、旋转、
        不透明度的变形以及随机数的部分，无需额外修改即可运行。当脚本中不包含
        AviUtl 专有写法时，此兼容层完全不介入(传统脚本的行为不变)。

        ○ 脚本头部指令的解释
                解释注释形式的参数声明，在执行前定义脚本主体所引用的变量。
                --dialog:项目名,local 变量=默认值;…
                        将各项目的声明部分(如 local 变量=默认值)展开到主体开头。
                        在 AviUtl 中这些是通过对话框编辑的值，此处以默认值运行。
                --color:0xRRGGBB
                        以指定颜色定义变量 color。
                --check0:项目名,默认值
                        以默认值定义 obj.check0 (0～3)。
                --param:任意 Lua 语句
                        将所写的 Lua 语句原样展开到主体开头。
                --track0: ～ --track3:
                        轨道条的值仍通过 obj.track0 ～ obj.track3 引用。头部的标签、
                        范围与默认值不会反映到 UI。

        ○ 通过 @ 的多区段
                当一个脚本以 @名称 定义了多个效果时，仅执行第一个区段。

        ○ obj.rand(st, ed [, seed [, frame]])
                与 AviUtl 兼容的确定性整数随机数。详情请参阅“obj 表 - 函数”。

        ○ 绘制·合成·加载
                支持 obj.draw / obj.drawpoly 的绘制合成、obj.copybuffer 的缓冲区操作、
                obj.load("figure"/"text"/"image"/"movie") 的生成与加载、obj.setfont、
                obj.effect、obj.getvalue 以及 obj.setoption / obj.getoption /
                obj.pixeloption。详情请参阅“obj 表 - 函数”。

        以下功能不在此兼容层的范围内。
        调用它们的脚本不会按预期运行。
                × 通过 require 加载模块，以及 io / dofile 等任意文件访问
                  (obj.load 的图像 / 视频加载已带文件头校验支持)。
                × obj.setanchor、scene.set / scene.get 等。


- 不支持项目·限制事项 -

        × 不支持 AviUtl2 的 obj.rand1()、obj.mes()、obj.setanchor() 等。
        × 不支持 AviUtl2 scene 表中的 scene.set()、scene.get() 等函数。
          （仅支持读取 scene.width 和 scene.height）
        × 不支持脚本进行任意文件访问（require、dofile、io 等）。
          仅 obj.load("image"/"movie") 会加载经文件头校验确认为真正媒体的文件。
        × 修改 obj.z 会反映到绘制中（包含在 ReadBack 中），
          但 3D 绘制支持有限。
        × 超过脚本超时时间（5 秒）时，执行将被强制终止。
          实现时需注意避免出现无限循环。
        × AviUtl2 的头部指令仅部分支持。--dialog / --color / --check / --param 的
          默认值会被解释（参阅“AviUtl 脚本的部分兼容”），但 --track@ 的标签、范围
          与默认值不会反映到 UI，值通过 Track0 ～ Track3 使用。
