- Acerca del efecto de script Lua -

        El efecto de video «Script Lua» de YukkuriMovieMaker4 utiliza
        el lenguaje Lua para los scripts.
        Los scripts se ejecutan en el runtime rápido incluido (LuaJIT) y,
        cuando este no está disponible, en MoonSharp (compatible con Lua 5.2).

        * Este efecto es una implementación propia y difiere de la especificación de scripts de AviUtl2.
        * Tiene una especificación distinta a los archivos de script de AviUtl2 (*.anm2, etc.).
        * Dado que IsAviUtlSupported = false, no se admite la salida EXO para AviUtl.


- Notas -

        o Los motores de ejecución son LuaJIT (predeterminado) y MoonSharp (compatible con Lua 5.2).
        o Las bibliotecas estándar disponibles se limitan a las siguientes:
            table, string, math, bit32
            * io, os, debug, ffi, require, etc. no están disponibles.
        o El tiempo de espera del script es de 5000 ms (5 segundos).
            Al agotarse el tiempo, la ejecución se termina forzosamente y se trata como error.
        o Los scripts no pueden acceder al sistema de archivos (incluido require).
        o No se admite la llamada a scripts externos desde dentro de un script.
        o Las operaciones de píxeles (getpixel/setpixel/getpixeldata) acceden a la imagen de
            entrada real, por lo que la transferencia de datos GPU→CPU solo ocurre en el
            primer acceso. Omitir estas llamadas cuando no sean necesarias acelera el procesamiento.


- Motores de ejecución y aceleración automática -

        Los scripts se analizan y se enrutan automáticamente al mejor motor.
        Los scripts sin directiva se ejecutan en el runtime rápido incluido
        (LuaJIT), y los mapeos por píxel autocontenidos se compilan además al
        núcleo CPU, aún más rápido. En ambos casos los resultados coinciden
        con MoonSharp.

        o Enrutamiento automático (cuando no hay directiva)
                - Un mapeo por píxel autocontenido (ver condiciones abajo)
                  -> núcleo CPU (rápido, exacto)
                - Todo lo demás -> Nativo (LuaJIT, solo si está desplegado)
                - Nativo no desplegado, o script vacío -> MoonSharp

        o Directivas de motor (un comentario en la primera línea del script)
                --!cpu       : Núcleo CPU. Compilado en tiempo de ejecución a un
                               árbol de expresiones y ejecutado en paralelo en
                               todos los núcleos de la CPU (idéntico a MoonSharp).
                --!gpu       : Núcleo GPU. Ejecutado como un sombreador de píxeles
                               de Direct2D, eliminando la transferencia GPU->CPU
                               para máxima velocidad. Los resultados pueden diferir
                               levemente por la precisión float32 (opcional).
                --!native    : LuaJIT (proceso aparte). Rápido e idéntico a
                               MoonSharp. Requiere la carpeta native desplegada.
                --!moonsharp : Fuerza el motor clásico MoonSharp.
                * Si una directiva no está disponible o no es compatible, recurre
                  automáticamente a un motor inferior (la salida nunca se rompe).

        o Sintaxis apta para el núcleo rápido (un mapeo por píxel puro)
                Solo un bucle anidado con la forma siguiente se convierte en núcleo.
                    for y = 0, obj.h - 1 do
                        for x = 0, obj.w - 1 do
                            local r, g, b, a = obj.getpixel(x, y)
                            obj.setpixel(x, y, r2, g2, b2 [, a2])
                        end
                    end
                - El cuerpo solo puede usar aritmética, math.*, comparaciones, el
                  modismo de umbral "cond and valor1 or valor2" y variables local.
                - Solo puede referenciar el píxel actual en proceso
                  (getpixel(x, y)) y valores como obj/scene/time.
                - Todo lo que use muestreo de vecinos (desenfoque, etc.),
                  escrituras en otras coordenadas, getpixeldata, getobject,
                  operaciones de tabla/cadena o sentencias if no se convierte en
                  núcleo y se ejecuta automáticamente en el motor clásico.

        Ejemplo (vuelve roja una figura blanca):
                --!gpu
                for y = 0, obj.h - 1 do
                    for x = 0, obj.w - 1 do
                        local r, g, b, a = obj.getpixel(x, y)
                        obj.setpixel(x, y, r, g / 2, b / 2, a)
                    end
                end


- Parámetros -

        El panel de efectos tiene los siguientes 4 controles deslizantes (con soporte de animación).
        Cada valor puede referenciarse en el script como obj.track0 ~ obj.track3.

        Track0 : Parámetro 0 (Predeterminado: 0, Rango: Mínimo ~ Máximo)
        Track1 : Parámetro 1 (Predeterminado: 0, Rango: Mínimo ~ Máximo)
        Track2 : Parámetro 2 (Predeterminado: 0, Rango: Mínimo ~ Máximo)
        Track3 : Parámetro 3 (Predeterminado: 0, Rango: Mínimo ~ Máximo)

        * Cada control deslizante es compatible con los deslizadores de animación (con interpolación).
        * Los archivos .lua pueden importarse/exportarse desde la barra de herramientas del editor de scripts.
        * Las casillas (obj.check0 - obj.check3) y el color (color) aparecen en el panel de
          efectos automáticamente cuando el script los referencia (las barras siempre se
          muestran). Este uso general no conlleva declaración de rango/valor predeterminado,
          así que gestione los rangos dentro del script. Las cabeceras --track / --check /
          --color de AviUtl siguen funcionando como antes.

        Hay 4 deslizadores numéricos, referenciados como obj.slider0 ~ obj.slider3.
        Los valores son enteros y no contienen decimales.

        Slider0 : Deslizador entero (Rango: 1 ~ 10)
        Slider1 : Deslizador entero (Rango: 0 ~ 100)
        Slider2 : Deslizador entero (Rango: 0 ~ 1000)
        Slider3 : Deslizador entero (Rango: -1000 ~ 1000)

        También hay parámetros de cadena, referenciados por los siguientes nombres.

        obj.text          : Cadena del campo de entrada de texto
        obj.font          : Nombre de fuente del selector de fuentes
        obj.dir           : Ruta del selector de carpetas
        obj.file_video    : Ruta de un archivo de video
        obj.file_audio    : Ruta de un archivo de audio
        obj.file_image    : Ruta de un archivo de imagen
        obj.file_project  : Ruta de un archivo de proyecto
        obj.file_mp4      : Ruta de un archivo MP4
        obj.file_exo      : Ruta de un archivo EXO
        obj.file_subtitle : Ruta de un archivo de subtítulos
        obj.file_shader   : Ruta de un archivo de sombreador

        * Los deslizadores y los parámetros de cadena aparecen en el panel de efectos
          automáticamente cuando el script los referencia.
        * Una ruta de archivo puede pasarse a obj.load para cargar una imagen o un video.
        * Estos parámetros están disponibles en los motores MoonSharp y nativo.


- Variables globales -

        Las siguientes variables pueden referenciarse directamente en el ámbito global del script.

        time          : Tiempo actual en segundos relativo al objeto [Solo lectura]
                        * Mismo valor que obj.time.
        frame         : Número de fotograma actual relativo al objeto [Solo lectura]
                        * Mismo valor que obj.frame.
        totalframe    : Número total de fotogramas del objeto [Solo lectura]
                        * Mismo valor que obj.totalframe.
        framerate     : Velocidad de fotogramas (fps) [Solo lectura]
                        * Mismo valor que obj.framerate.
        timelineframe : Número de fotograma actual relativo a toda la línea de tiempo [Solo lectura]
                        * obj.frame es un fotograma relativo desde el inicio del objeto,
                          mientras que este es el fotograma absoluto desde el inicio de la línea de tiempo.
        timelinetime  : Tiempo actual en segundos relativo a toda la línea de tiempo [Solo lectura]
        layer         : Número de capa donde se encuentra el objeto [Solo lectura]
                        * Mismo valor que obj.layer.
        obj           : Tabla de información del objeto (ver «Tabla obj» a continuación)
        scene         : Tabla de información de la escena (ver «Tabla scene» a continuación)
        ymm4          : YMM4 specific meta-information table (see "ymm4 Table" below)
        anim          : Animation utility table (see "anim Table" below)


- Tabla scene -

        scene.width   : Resolución horizontal (píxeles) de la escena (proyecto) [Solo lectura]
        scene.height  : Resolución vertical (píxeles) de la escena (proyecto) [Solo lectura]

        scene.set(name, value)
                Guarda un valor en un área compartida a nivel de escena.
                El valor puede leerse con scene.get(name) desde todos los efectos de
                script Lua de la misma escena, a través de elementos y fotogramas.
                name  : Nombre del valor (cadena)
                value : Valor a guardar (número, cadena o booleano). Pasar nil lo elimina.
                * Los valores que no sean números, cadenas o booleanos (tablas,
                  funciones, etc.) no se guardan.
                * Los nombres y los valores de cadena se truncan más allá de 4095
                  bytes en UTF-8.
                Ejemplo: scene.set("shared_x", obj.x)
        scene.get(name)
                Obtiene un valor guardado con scene.set(). Devuelve nil para nombres
                no establecidos.
                Ejemplo: local x = scene.get("shared_x") or 0
        * Los valores escritos por su propio script pueden leerse inmediatamente con
          scene.get dentro de la misma ejecución. Los valores escritos por otros
          elementos son visibles a partir del fotograma siguiente, por lo que el
          resultado no depende del orden de evaluación dentro de un fotograma.
        * Cuando un script termina con un error o por tiempo de espera, ninguna de
          sus escrituras se aplica al área compartida.
        * Al iniciar la exportación de video, el área compartida comienza vacía, por
          lo que exportar el mismo proyecto siempre produce el mismo resultado.
        * Se pueden guardar hasta 4096 nombres por escena. Las escrituras a nombres
          nuevos por encima del límite se ignoran.
        * Los valores son independientes por escena y se mantienen en memoria hasta
          que YMM4 se cierra. No se guardan en el archivo del proyecto.


- Tabla obj - Variables -

        Las siguientes variables contienen la información de renderizado actual del objeto.
        * Las variables no marcadas como Solo lectura pueden modificarse en el script y los cambios se reflejan en el renderizado.

        o Coordenadas / Desplazamiento
        obj.x         : Coordenada X de referencia de visualización (modificable)
                        * Equivale a «Coordenada X de referencia de visualización» en la especificación de AviUtl2.
                        * Internamente se establece desde DrawDescription.Draw.X;
                          los cambios surten efecto después de ejecutar el script.
        obj.y         : Coordenada Y de referencia de visualización (modificable)
                        * Internamente se establece desde DrawDescription.Draw.Y.
        obj.z         : Coordenada Z de referencia de visualización (modificable)
                        * Internamente se establece desde DrawDescription.Draw.Z.
        obj.ox        : Desplazamiento de centro X (modificable)
                        * Internamente se establece desde DrawDescription.CenterPoint.X (inDesc).
                        * Equivale a «Coordenada X relativa desde la referencia (obj.ox)» en AviUtl2,
                          pero en este plugin se trata como CenterPoint (desplazamiento del punto central).
        obj.oy        : Desplazamiento de centro Y (modificable)
                        * Internamente se establece desde DrawDescription.CenterPoint.Y (inDesc).

        o Escala / Transformación
        obj.zoom      : Factor de escala (modificable)
                        * 1.0 es la escala normal (no 0.0), compatible con AviUtl2.
                        * Internamente se calcula como el promedio de ZoomX y ZoomY (zoomAvg).
                        * En la salida, se expande al zoom individual de X/Y como obj.zoom * (1 ± obj.aspect).
        obj.aspect    : Relación de aspecto (modificable), Rango: -1.0 ~ 1.0
                        * Los valores positivos reducen en la dirección X; los negativos en la dirección Y.
                        * Internamente se calcula como (ZoomX - ZoomY) / (ZoomX + ZoomY).
        obj.sx        : Scale factor in X direction (writable) 1.0 = 100%
                        * Precise independent scale control specific to YMM4.
                        * Takes precedence over zoom/aspect modifications.
        obj.sy        : Scale factor in Y direction (writable) 1.0 = 100%
                        * Precise independent scale control specific to YMM4.
                        * Takes precedence over zoom/aspect modifications.
        obj.alpha     : Opacidad (modificable), Rango: 0.0 ~ 255.0
                        * AviUtl2 usa 0.0~1.0, pero este plugin usa 0.0~255.0.
                        * Internamente se establece desde DrawDescription.Opacity * 255.
                        * Se divide por 255 y se normaliza a 0.0~1.0 en la salida (con Clamp).
                        * El script predeterminado establece obj.alpha = math.min(time * 255, 255),
                          proporcionando una implementación de fundido de entrada predeterminada.

        o Rotación
        obj.rx        : Ángulo de rotación alrededor del eje X (modificable), Unidad: grados (°), 360.0 = una vuelta completa
                        * Internamente se establece desde DrawDescription.Rotation.X.
        obj.ry        : Ángulo de rotación alrededor del eje Y (modificable), Unidad: grados (°), 360.0 = una vuelta completa
                        * Internamente se establece desde DrawDescription.Rotation.Y.
        obj.rz        : Ángulo de rotación alrededor del eje Z (modificable), Unidad: grados (°), 360.0 = una vuelta completa
                        * Internamente se establece desde DrawDescription.Rotation.Z.
        obj.rxr       : Rotation angle around X axis in radians (writable)
                        * Takes precedence if rx is also modified simultaneously.
        obj.ryr       : Rotation angle around Y axis in radians (writable)
                        * Takes precedence if ry is also modified simultaneously.
        obj.rzr       : Rotation angle around Z axis in radians (writable)
                        * Takes precedence if rz is also modified simultaneously.

        o Tamaño de imagen
        obj.w         : Ancho de la imagen de entrada en píxeles [Solo lectura]
                        * Internamente es un entero derivado de D2D GetImageLocalBounds.
        obj.h         : Alto de la imagen de entrada en píxeles [Solo lectura]
                        * Internamente es un entero derivado de D2D GetImageLocalBounds.
        obj.hw        : Half of the input image width (= obj.w / 2) [ReadOnly]
        obj.hh        : Half of the input image height (= obj.h / 2) [ReadOnly]
        obj.diagonal  : Longitud diagonal de la imagen de entrada (= math.sqrt(w*w + h*h)) [ReadOnly]
        obj.cx        : Coordenada X central de la imagen (= obj.w / 2) [Solo lectura]
                        * Coordenada central de referencia para operaciones de píxeles.
        obj.cy        : Coordenada Y central de la imagen (= obj.h / 2) [Solo lectura]

        o Tiempo / Fotograma
        obj.time      : Tiempo actual en segundos relativo al objeto [Solo lectura]
                        * Mismo valor que la variable global time.
                        * Internamente se establece desde EffectDescription.ItemPosition.Time.TotalSeconds.
        obj.frame     : Número de fotograma actual relativo al objeto [Solo lectura]
                        * Mismo valor que la variable global frame.
                        * Internamente se establece desde EffectDescription.ItemPosition.Frame.
        obj.totalframe: Número total de fotogramas del objeto [Solo lectura]
                        * Mismo valor que la variable global totalframe.
                        * Internamente se establece desde EffectDescription.ItemDuration.Frame.
        obj.framerate : Velocidad de fotogramas (fps) [Solo lectura]
                        * Mismo valor que la variable global framerate.
                        * Internamente se establece desde EffectDescription.FPS.

        o Controles deslizantes
        obj.track0    : Valor actual de Track0 en el panel de efectos [Solo lectura]
                        * Valor tras la interpolación de animación.
        obj.track1    : Valor actual de Track1 en el panel de efectos [Solo lectura]
        obj.track2    : Valor actual de Track2 en el panel de efectos [Solo lectura]
        obj.track3    : Valor actual de Track3 en el panel de efectos [Solo lectura]

        o Deslizadores
        obj.slider0   : Valor actual de Slider0 en el panel de efectos (entero) [Solo lectura]
        obj.slider1   : Valor actual de Slider1 en el panel de efectos (entero) [Solo lectura]
        obj.slider2   : Valor actual de Slider2 en el panel de efectos (entero) [Solo lectura]
        obj.slider3   : Valor actual de Slider3 en el panel de efectos (entero) [Solo lectura]

        o Parámetros de cadena
        obj.text      : Cadena del campo de entrada de texto [Solo lectura]
        obj.font      : Nombre de fuente del selector de fuentes [Solo lectura]
        obj.dir       : Ruta del selector de carpetas [Solo lectura]
        obj.file_video ~ obj.file_shader : Ruta de cada selector de archivos [Solo lectura]

        o Capa
        obj.layer     : Número de capa donde se encuentra el objeto [Solo lectura]
                        * Mismo valor que la variable global layer.
                        * Internamente se establece desde EffectDescription.Layer.


- Tabla obj - Cambios de variables y reflejo en el renderizado -

        Tras la ejecución del script, solo las siguientes variables se leen de vuelta al renderizado (se aplican).

        Variables aplicadas:
            obj.x, obj.y, obj.z
            obj.ox, obj.oy
            obj.zoom, obj.aspect
            obj.alpha
            obj.rx, obj.ry, obj.rz

        * obj.w, obj.h, obj.cx, obj.cy, obj.frame, obj.time,
          obj.totalframe, obj.framerate, obj.track0 ~ obj.track3,
          obj.layer son Solo lectura y los cambios no se reflejan en el renderizado.

        Fórmulas de valores de salida:
            ZoomX = obj.zoom * (1.0 + obj.aspect)
            ZoomY = obj.zoom * (1.0 - obj.aspect)
            Opacity = Clamp(obj.alpha / 255.0, 0.0, 1.0)
            Draw = (obj.x, obj.y, obj.z)
            CenterPoint = (obj.ox, obj.oy)
            Rotation = (obj.rx, obj.ry, obj.rz)


- Tabla obj - Funciones -

        o obj.getobject(tag [, frame])
                Obtiene la información de dibujo de otro objeto que tiene la
                etiqueta especificada.
                * La etiqueta coincide con el texto introducido en el campo
                  "Observación" del elemento de destino.
                * Si se indica el segundo argumento frame, se obtienen los
                  valores en ese fotograma de la línea de tiempo. Si se omite,
                  se usa el fotograma actual.
                * Si varios elementos comparten la misma etiqueta, se devuelve
                  con prioridad el elemento visible en el fotograma de destino.
                  Si ninguno es visible, se devuelve el primer elemento que
                  coincida.
                * Los valores devueltos son las coordenadas, rotación y escala
                  propias del elemento. NO se incluyen las transformaciones por
                  control de grupo, control de cámara o efectos superiores.
                * Cada valor usa el mismo sistema de unidades que la tabla obj.
                tag     : El texto definido en la observación del elemento de destino
                frame   : El fotograma absoluto de la línea de tiempo (opcional)
                Retorno : Una tabla de información del objeto. Devuelve nil cuando
                          no existe ningún elemento coincidente.
                          .exist : Si es visible en el fotograma de destino (boolean)
                          .x     : Coordenada base de visualización X
                          .y     : Coordenada base de visualización Y
                          .z     : Coordenada base de visualización Z
                          .zoom  : Escala (1.0 = tamaño original)
                          .sx    : Escala en X (= zoom)
                          .sy    : Escala en Y (= zoom)
                          .rx    : Ángulo de rotación en el eje X (grados, siempre 0)
                          .ry    : Ángulo de rotación en el eje Y (grados, siempre 0)
                          .rz    : Ángulo de rotación en el eje Z (grados)
                          .rxr   : Ángulo de rotación en el eje X (radianes, siempre 0)
                          .ryr   : Ángulo de rotación en el eje Y (radianes, siempre 0)
                          .rzr   : Ángulo de rotación en el eje Z (radianes)
                          .alpha : Opacidad (0.0 - 255.0, fundido aplicado)
                          .layer : Número de capa donde está colocado
                Ejemplo:
                    local funnel = obj.getobject("funnel")
                    if funnel then
                        obj.x = funnel.x
                        obj.y = funnel.y
                    end
                Fijar a la posición de disparo (la observación del emisor es "funnel"):
                    local fire = timelineframe - frame
                    local src = obj.getobject("funnel", fire)
                    if src then
                        obj.x = src.x
                        obj.y = src.y
                    end

        o obj.getpixel(x, y)
                Obtiene información del píxel en las coordenadas especificadas.
                * Se produce una transferencia de datos de imagen GPU→CPU en la primera llamada (con sobrecarga de procesamiento).
                * Las coordenadas de píxel usan la esquina superior izquierda de la imagen como (0, 0), con derecha y abajo como positivos.
                * Los valores devueltos son equivalentes al alfa directo convertido desde el alfa premultiplicado.
                x       : Coordenada X (0 ~ obj.w - 1)
                y       : Coordenada Y (0 ~ obj.h - 1)
                Devuelve: 4 valores r, g, b, a (valores de retorno múltiple)
                          r = Componente rojo (0.0 ~ 255.0)
                          g = Componente verde (0.0 ~ 255.0)
                          b = Componente azul (0.0 ~ 255.0)
                          a = Valor alfa (0.0 ~ 255.0)
                          * Devuelve (0, 0, 0, 0) para coordenadas fuera de rango.
                Ejemplo: r, g, b, a = obj.getpixel(10, 20)

        o obj.setpixel(x, y, r, g, b [, a])
                Sobrescribe la información del píxel en las coordenadas especificadas.
                * Los datos de píxel se marcan automáticamente como «modificados» al llamar,
                  y la imagen modificada se emite después de que el script termina.
                * Se almacena internamente en formato alfa premultiplicado (BGRA).
                  Se realiza una conversión premultiplicada en setpixel.
                x       : Coordenada X (0 ~ obj.w - 1)
                y       : Coordenada Y (0 ~ obj.h - 1)
                r       : Componente rojo (0.0 ~ 255.0)
                g       : Componente verde (0.0 ~ 255.0)
                b       : Componente azul (0.0 ~ 255.0)
                a       : Valor alfa (predeterminado: 255.0)
                Ejemplo: obj.setpixel(10, 20, 255, 0, 0, 128)

        o obj.getpixeldata()
                Devuelve un objeto proxy para acceso masivo a todos los datos de píxeles.
                * Manipulación de píxeles más rápida en comparación con getpixel/setpixel.
                * Se produce una transferencia de datos de imagen GPU→CPU en la primera llamada (con sobrecarga de procesamiento).
                Devuelve: Objeto PixelDataProxy
                          .width  : Ancho de la imagen en píxeles (= obj.w)
                          .height : Alto de la imagen en píxeles (= obj.h)
                          :get(index)        : Obtiene el valor del canal por índice
                          :set(index, value) : Establece el valor del canal por índice

                Especificación de índice:
                    Los índices empiezan en 1 (siguiendo la convención de Lua).
                    El orden de canales es R, G, B, A repetido.
                    Índice del canal R para pixel(x, y) =
                        (y * obj.w + x) * 4 + 1
                    Índice del canal G para pixel(x, y) =
                        (y * obj.w + x) * 4 + 2
                    Índice del canal B para pixel(x, y) =
                        (y * obj.w + x) * 4 + 3
                    Índice del canal A para pixel(x, y) =
                        (y * obj.w + x) * 4 + 4

                    * Modificar el alfa (canal 4) también convertirá RGB proporcionalmente.
                    * Modificar los canales RGB aplicará una conversión premultiplicada.
                    * Los valores obtenidos son equivalentes al alfa directo, igual que get(index).

                Ejemplo:
                    local pd = obj.getpixeldata()
                    local idx = (10 * obj.w + 20) * 4 + 1  -- Canal R de (x=20, y=10)
                    local r = pd:get(idx)
                    pd:set(idx, 128)

        o obj.putpixeldata()
                Una función vacía (sin operación) definida por compatibilidad.
                * En los scripts de AviUtl se usan getpixeldata/putpixeldata juntos,
                  pero en este plugin los cambios mediante setpixel/set se aplican automáticamente,
                  por lo que no es necesario llamar a esta función.

        o obj.rand(st, ed [, seed [, frame]])
                Devuelve un valor aleatorio entero determinista dentro del rango st ~ ed.
                * Con el mismo seed y frame, siempre devuelve el mismo valor.
                * Si se omite seed se trata como 0; si se omite frame se usa el
                  fotograma actual (obj.frame).
                * Devuelve el mismo valor en el motor MoonSharp y en el runtime LuaJIT.
                  Para un valor aleatorio decimal independiente del motor, use anim.rand.
                st      : Límite inferior del rango (truncado a entero)
                ed      : Límite superior del rango (truncado a entero)
                seed    : Semilla aleatoria (opcional, predeterminado 0)
                frame   : Fotograma usado para la variación (opcional, predeterminado obj.frame)
                Devuelve: Un entero dentro de st ~ ed
                Ejemplo: obj.ox = obj.ox + obj.rand(-10, 10, 1, obj.frame)

        o obj.load(target, ...)
                Reemplaza el búfer del objeto con contenido generado. Se admiten los
                targets "figure", "text", "image" (archivo de imagen) y "movie"
                (archivo de vídeo). Tras la carga, obj.w / obj.h, etc. se actualizan al
                tamaño generado y pueden leerse con getpixel justo después de la carga.
                Funciona en el motor MoonSharp y en el motor rápido (LuaJIT).

                obj.load("figure", name, color, size [, line [, aspect]])
                        Genera una forma y reemplaza el búfer (centrada).
                        name   : 円 / 四角形 / 三角形 / 五角形 / 六角形 / 星形
                                 (también los alias en inglés circle, rect, triangle,
                                 pentagon, hexagon, star).
                        color  : Color de relleno 0xRRGGBB.
                        size   : Tamaño base en píxeles.
                        line   : Grosor del contorno (opcional). 0 rellena la forma.
                        aspect : Distorsión -1.0 ~ 1.0 (opcional).
                        Ejemplo: obj.load("figure", "円", 0xff3366, 200)

                obj.load("text", cadena)
                        Dibuja la cadena con la fuente, el tamaño y el color definidos
                        por obj.setfont y reemplaza el búfer (con DirectWrite).
                        Ejemplo: obj.setfont("Yu Gothic UI", 48); obj.load("text", "字幕")

                obj.load("image", ruta)
                        Carga un archivo de imagen y reemplaza el búfer.
                        * Por seguridad se valida la cabecera inicial y solo se cargan
                          imágenes genuinas (PNG / JPEG / GIF / BMP / TIFF / WebP).
                        Ejemplo: obj.load("image", "C:\\img\\logo.png")

                obj.load("movie", ruta [, tiempo])
                        Carga el fotograma en el tiempo indicado de un archivo de vídeo
                        y reemplaza el búfer (decodificado con Media Foundation).
                        tiempo : Segundos. Por defecto obj.time si se omite.
                        * Por seguridad se valida la cabecera inicial y solo se cargan
                          vídeos genuinos (MP4 / MOV / MKV / WebM / AVI / FLV / ASF).
                        Ejemplo: obj.load("movie", "C:\\mov\\clip.mp4", obj.time)

        o obj.setfont(fuente [, tamaño [, estilo [, color]]])
                Establece la fuente usada por obj.load("text").
                estilo : Bits. 1=negrita, 2=cursiva, 3=negrita+cursiva.
                color  : 0xRRGGBB.
                Ejemplo: obj.setfont("Meiryo", 64, 1, 0xffffff)

        o obj.draw([ox, oy, oz [, zoom [, alpha [, aspect]]]])
                Compone la imagen actual del objeto en el búfer de acumulación.
                Llamarla varias veces dibuja copias superpuestas (estelas,
                partículas, etc.).
                ox, oy, oz : Desplazamiento desde el centro (por defecto 0).
                zoom       : 1.0 es tamaño original (por defecto 1.0).
                alpha      : 0.0 ~ 1.0 (por defecto 1.0).
                aspect     : -1.0 ~ 1.0 (por defecto 0).
                * Una vez llamada, se suprime el dibujo estándar y se emite el
                  resultado compuesto.

        o obj.drawpoly(x0,y0,z0, x1,y1,z1, x2,y2,z2, x3,y3,z3
                        [, u0,v0, u1,v1, u2,v2, u3,v3 [, alpha]])
                Aplica la imagen del objeto como textura sobre un cuadrilátero dado por
                4 vértices. El orden es superior-izq, superior-der, inferior-der,
                inferior-izq. Si se omiten las UV se usa toda la imagen.
                * Si solo se da un 13.º argumento (UV omitidas), se usa como alpha
                  (12 coordenadas + alpha).
                * Se mapea con una transformación afín (exacta para paralelogramos,
                  aproximada para no paralelogramos).

        o obj.copybuffer(destino, origen)
                Copia una imagen entre búferes, seleccionado por el primer carácter.
                  "obj"          : Búfer de imagen del objeto
                  "tmp"          : Búfer temporal
                  "cache:nombre" : Caché con nombre (se conserva entre fotogramas)
                Ejemplo: obj.copybuffer("tmp", "obj")  -- guardar
                         obj.copybuffer("obj", "tmp")  -- restaurar
                * "frm" (búfer de fotograma / fondo) no puede obtenerse por la
                  estructura de YMM4.

        o obj.getvalue(target)
                Obtiene el valor actual de obj (igual que obj.x, obj.track0, etc.).
                target : "x","y","z","ox","oy","rx","ry","rz","zoom","aspect",
                         "alpha","sx","sy","track0"~"track3","time","frame", etc.
                Devuelve: un número (0 para un target desconocido).

        o obj.setoption(nombre, valor) / obj.getoption(nombre)
                Establece / obtiene una opción. Las opciones no admitidas se aceptan
                para no detener el script (getoption devuelve el valor guardado, 0 si
                no está establecido).
                Opciones que afectan al dibujo actualmente:
                  "antialias" : 0 = vecino más cercano, en otro caso interpolación
                                lineal (obj.draw / obj.drawpoly).
                  "blend" : número de modo de fusión (obj.draw / obj.drawpoly). El
                    mismo orden (empezando en 0) que la lista desplegable "Modo de
                    fusión" del panel de propiedades de YMM4:
                    0=Normal,1=Síntesis de interpolación,2=Comparación (oscura),
                    3=Multiplicar,4=Color de relleno,5=Burn-in (lineal),
                    6=Comparación (clara),7=Pantalla,8=Color sin máscara,
                    9=Sin máscara (lineal) - Aditivo,10=Adición,11=Superponer,
                    12=Luz suave,13=Luz dura,14=Luz viva,15=Luz lineal,
                    16=Pin Light,17=Mezcla dura,18=diferencia,19=Excluido,
                    20=Sustracción,21=División,22=Tono,23=Saturación,24=Color,
                    25=Luminosidad,26=Comparación de colores (claro),27=Fondo,
                    28=Comparación de colores (oscuro),29=Supresión,
                    30=Recorte en el fondo,31=Sólo partes no superpuestas,
                    32=Máscara de inversión de color. Los valores fuera de rango
                    o no válidos se tratan como 0 (Normal).
                  "draw_state" : un booleano que decide si este efecto emite su
                    imagen de trabajo. true emite la imagen de trabajo; false deja
                    pasar la entrada sin cambios. Sirve para emitir la imagen de
                    trabajo sin obj.draw ni setpixel, o para suprimir la salida.
                    Funciona en ambos motores.
                  "drawtarget" : cambia el destino de obj.draw / obj.drawpoly.
                    "framebuffer" (por defecto) compone en el búfer de fotograma.
                    "tempbuffer" compone en el búfer de trabajo tmp en el momento. En
                    obj.setoption("drawtarget", "tempbuffer", w, h) los valores w, h
                    dan el tamaño con que se inicializa el búfer tmp; omítelos para
                    conservarlo. Recupera el resultado de tmp a la imagen de trabajo
                    con obj.copybuffer("obj", "tmp"). La composición en tempbuffer es
                    normal (source-over); blend se aplica al dibujo en framebuffer.
                    Funciona en ambos motores. En MoonSharp la composición en
                    tempbuffer se ejecuta en la GPU y cambia automáticamente a la
                    composición por CPU cuando la GPU no está disponible. Por el
                    redondeo, el resultado puede diferir muy ligeramente entre los
                    motores.

        o obj.pixeloption(nombre, valor)
                Guarda como estado una opción para las operaciones de píxeles.

        o obj.setanchor(nombre, cantidad [, opción, ..])
                Muestra puntos de anclaje en la vista previa para ajustar sus
                coordenadas arrastrando con el ratón. Las coordenadas ajustadas se
                guardan con el proyecto.
                nombre : el nombre de la variable Lua que recibe las coordenadas,
                    como cadena. La variable se dispone como pos={x0,y0,x1,y1,…}, o
                    {x0,y0,z0,x1,y1,z1,…} con "xyz". Las coordenadas son relativas
                    al centro del objeto.
                cantidad : el número de puntos de anclaje. Hasta 32.
                opción : se enumeran como cadenas.
                    "line" : une los puntos de anclaje con líneas.
                    "loop" : une los puntos de anclaje y cierra el bucle.
                    "star" : une cada punto de anclaje con el centro del objeto.
                    "arm"  : une el centro del objeto con cada punto en orden.
                    "xyz"  : trata los puntos como coordenadas 3D. 2D por defecto.
                Devuelve el número de puntos de anclaje. Funciona en ambos motores.
                El modo de barra "track" aún no se admite y devuelve 0.
                Ejemplo:
                  num = obj.track0
                  obj.setanchor("pos", num, "loop")
                  for i = 0, num - 1 do
                    obj.draw(pos[i * 2 + 1], pos[i * 2 + 2])
                  end

- Ejemplos de script -

        o Fundido de entrada (script predeterminado)
                obj.alpha = math.min(time * 255, 255)

        o Fundido de salida
                obj.alpha = math.max((obj.totalframe - obj.frame) / obj.framerate * 255, 0)

        o Controlar la opacidad con Track0
                obj.alpha = obj.track0

        o Invertir el componente rojo de los píxeles

                    for x = 0, obj.w - 1 do
                        local r, g, b, a = obj.getpixel(x, y)
                        obj.setpixel(x, y, 255 - r, g, b, a)
                    end
                end

        o Conversión a escala de grises usando getpixeldata (rápido)
                local pd = obj.getpixeldata()
                local w = pd.width
                local h = pd.height
                for y = 0, h - 1 do
                    for x = 0, w - 1 do
                        local base = (y * w + x) * 4
                        local r = pd:get(base + 1)
                        local g = pd:get(base + 2)
                        local b = pd:get(base + 3)
                        local a = pd:get(base + 4)
                        local gray = r * 0.299 + g * 0.587 + b * 0.114
                        pd:set(base + 1, gray)
                        pd:set(base + 2, gray)
                        pd:set(base + 3, gray)
                    end
                end

- Compatibilidad parcial con scripts de AviUtl -

        Una capa de compatibilidad permite ejecutar, sin reescritura, las partes de
        los scripts de efecto de animación de AviUtl (la familia *.anm) relativas a
        transformación, opacidad y números aleatorios. Cuando un script no contiene
        notación específica de AviUtl, esta capa no interviene en absoluto (los
        scripts existentes se comportan exactamente igual que antes).

        o Directivas de encabezado del script
                Las declaraciones de parámetros escritas como comentarios se
                interpretan, y las variables que el cuerpo del script referencia se
                definen antes de la ejecución.
                --dialog:etiqueta,local var=predeterminado;...
                        Expande la parte de declaración de cada elemento
                        (como local var=predeterminado) al inicio del cuerpo.
                        En AviUtl se editan en un cuadro de diálogo; aquí se usan
                        los valores predeterminados.
                --color:0xRRGGBB
                        Define la variable color con el color especificado.
                --check0:etiqueta,predeterminado
                        Define obj.check0 (0 a 3) con el valor predeterminado.
                --param:<sentencia Lua>
                        Expande la sentencia Lua escrita al inicio del cuerpo.
                --track0: a --track3:
                        Los valores de las barras de seguimiento se referencian
                        mediante obj.track0 a obj.track3 como antes. La etiqueta, el
                        rango y el valor predeterminado del encabezado no se reflejan en la UI.

        o Múltiples secciones con @
                Cuando un único script define varios efectos con @nombre, solo se
                ejecuta la primera sección.

        o obj.rand(st, ed [, seed [, frame]])
                Función de número aleatorio entero determinista compatible con AviUtl.
                Consulte "Tabla obj - Funciones" para más detalles.

        o Dibujo, composición y carga
                Se admiten el dibujo y composición con obj.draw / obj.drawpoly, las
                operaciones de búfer con obj.copybuffer, la generación y carga con
                obj.load("figure"/"text"/"image"/"movie"), obj.setfont, obj.effect,
                obj.getvalue y obj.setoption / obj.getoption / obj.pixeloption.
                Consulte "Tabla obj - Funciones" para más detalles.

        o Mapa de compatibilidad AviUtl / AviUtl2 para obj.effect
                obj.effect("nombre de efecto", "nombre de parámetro", valor, ...)
                consulta primero el mapa de compatibilidad. Los efectos declarados como
                nombres de AviUtl / AviUtl2 pasan por la conversión de
                escala/desplazamiento declarada antes de llegar a las propiedades de
                YMM4. Los nombres no declarados en el mapa se resuelven directamente por
                el nombre de efecto o la palabra clave de YMM4, por lo que las llamadas
                existentes siguen funcionando. El mapa se construye como la inversa de
                la propia exportación de AviUtl de YMM4 (salida .exo), por lo que es
                reversible de forma exacta. El motor de destino se selecciona con
                --!aviutl (predeterminado) / --!aviutl2.
                Los nombres de efecto declarados en el mapa y sus parámetros principales
                son los siguientes.
                  ぼかし: 範囲, サイズ固定
                  境界ぼかし: 範囲
                  方向ブラー: 範囲, 角度, サイズ固定
                  放射ブラー: 範囲, X, Y, サイズを固定
                  閃光: 範囲, X, Y, サイズを固定
                  クロマキー: 色相範囲, 境界補正
                  クリッピング: 上, 下, 左, 右, 中心の位置を変更
                  斜めクリッピング: 中心X, 中心Y, 角度, ぼかし, 幅
                  領域拡張: 上, 下, 左, 右
                  座標: X, Y, Z
                  回転: X, Y, Z
                  リサイズ: 拡大率, X, Y, 補間なし
                  画像ループ: 横回数, 縦回数
                  極座標変換: 中心幅, 渦巻
                  グラデーション: 強さ, 中心X, 中心Y, 角度, 幅
                  単色化: 強さ, 輝度を保持する
                  縁取り: サイズ, ぼかし
                  反転: -
                  モザイク: サイズ
                  ディスプレイスメントマップ: ぼかし
                  動画ファイル合成: X, Y, 拡大率
                  色調補正: 明るさ, ｺﾝﾄﾗｽﾄ, 色相, 輝度, 彩度
                  色ずれ: ずれ幅, 角度, 強さ
                色調補正 y 色ずれ corresponden a efectos del complemento de la comunidad.

        Lo siguiente queda fuera del alcance de esta capa. Los scripts que lo llamen no
        se comportarán como se espera.
                x Carga de módulos mediante require y acceso arbitrario a archivos como
                  io / dofile (la carga de imagen / vídeo con obj.load sí se admite con
                  validación de cabecera).
                x obj.setanchor, etc.


- Funciones no admitidas / Limitaciones -

        x obj.rand1(), obj.mes(), obj.setanchor(), etc. de AviUtl2 no son compatibles.
        x El acceso arbitrario a archivos desde scripts (require, dofile, io, etc.) no es compatible.
          Solo obj.load("image"/"movie") carga archivos confirmados como medios genuinos
          mediante validación de la cabecera inicial.
        x Modificar obj.z se refleja en el renderizado (incluido en ReadBack),
          pero el soporte de renderizado 3D es limitado.
        x Si se supera el tiempo de espera del script (5 segundos), la ejecución se termina forzosamente.
          Se debe tener cuidado en la implementación para evitar bucles infinitos.
        x Las directivas de encabezado de AviUtl2 solo son compatibles parcialmente. Los valores
          predeterminados de --dialog / --color / --check / --param se interpretan (consulte
          "Compatibilidad parcial con scripts de AviUtl"), pero la etiqueta, el rango y el valor
          predeterminado de --track@ no se reflejan en la UI; los valores se usan vía Track0 ~ Track3.
