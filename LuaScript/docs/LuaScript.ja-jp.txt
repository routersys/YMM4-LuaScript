- Luaスクリプトエフェクトについて -

        YukkuriMovieMaker4 の「Luaスクリプト」映像エフェクトで
        利用可能なスクリプトには Lua 言語を使用します。
        スクリプトは同梱の高速ランタイム(LuaJIT)で実行され、
        利用できない環境では MoonSharp (Lua 5.2 互換) で実行されます。

        ※このエフェクトは AviUtl2 のスクリプト仕様とは異なる独自実装です。
        ※AviUtl2 のスクリプトファイル(*.anm2 等)とは別仕様です。
        ※IsAviUtlSupported = false のため、AviUtl 向けの EXO 出力には対応しません。


- 注意点 -

        ○スクリプトの実行エンジンは LuaJIT(標準)と MoonSharp (Lua 5.2 互換) です。
        ○利用可能な標準ライブラリは以下に限定されます。
            table, string, math, bit32
            ※io, os, debug, ffi, require 等は使用できません。
        ○スクリプトのタイムアウトは 5000ms (5秒) です。
            タイムアウト時は実行が強制終了され、エラーとして扱われます。
        ○スクリプトからファイルへのアクセスはできません (require を含む)。
        ○スクリプト内からの外部スクリプト呼び出しには対応していません。
        ○ピクセル操作(getpixel/setpixel/getpixeldata)は実際の入力画像に
            アクセスするため、初回アクセス時にのみ GPU→CPU へのデータ転送が
            発生します。不要な場合は呼び出しを省略することで処理を高速化できます。


- 実行エンジンと自動高速化 -

        スクリプトは構文を解析して自動的に最適なエンジンへ振り分けられます。
        ディレクティブ無しのスクリプトは同梱の高速ランタイム(LuaJIT)で
        実行され、自ピクセル完結のピクセル写像はさらに高速な CPU カーネルへ
        変換されます。いずれの結果も MoonSharp と一致します。

        ○自動振り分け(ディレクティブ無しのとき)
                ・自ピクセル完結のピクセル写像(下記条件) → CPU カーネル(高速・厳密)
                ・上記以外 → ネイティブ(LuaJIT, 配備時のみ)
                ・native 未配備または空のスクリプト → MoonSharp

        ○エンジン指定ディレクティブ(スクリプト先頭行のコメント)
                --!cpu       : CPU カーネル。式ツリーへ実行時コンパイルして
                               全 CPU コアで並列実行します(MoonSharp と完全一致)。
                --!gpu       : GPU カーネル。Direct2D ピクセルシェーダとして実行し、
                               GPU→CPU 転送を排して最速です。float32 精度のため
                               結果はごく僅かに異なり得ます(オプトイン)。
                --!native    : LuaJIT(別プロセス)。高速かつ MoonSharp と一致。
                               native フォルダの配備が必要です。
                --!moonsharp : 従来の MoonSharp を強制します。
                ※ 指定が利用不可/非対応の場合は、自動的に下位エンジンへ
                  フォールバックします(出力が壊れることはありません)。

        ○高速カーネルが成立する構文(純粋ピクセル写像)
                次の形の二重ループのみがカーネル化されます。
                    for y = 0, obj.h - 1 do
                        for x = 0, obj.w - 1 do
                            local r, g, b, a = obj.getpixel(x, y)
                            obj.setpixel(x, y, r2, g2, b2 [, a2])
                        end
                    end
                ・本体で使えるのは四則演算・math.*・比較・しきい値の
                  「条件 and 値1 or 値2」・local 変数のみです。
                ・参照できるのは処理中の自ピクセル(getpixel(x, y))と
                  obj/scene/time 等の値だけです。
                ・近傍参照(ぼかし等)、別座標への書き込み、getpixeldata、
                  getobject、テーブル/文字列操作、if 文などを含む場合は
                  カーネル化されず、自動的に従来エンジンで実行されます。

        例(白い図形が赤くなる):
                --!gpu
                for y = 0, obj.h - 1 do
                    for x = 0, obj.w - 1 do
                        local r, g, b, a = obj.getpixel(x, y)
                        obj.setpixel(x, y, r, g / 2, b / 2, a)
                    end
                end


- パラメータ -

        エフェクトパネルには以下の 4 つのトラックバー(アニメーション対応)があります。
        各値はスクリプト内で obj.track0 ～ obj.track3 として参照できます。

        Track0 : パラメータ0 (デフォルト: 0, 範囲: 最小値 ～ 最大値)
        Track1 : パラメータ1 (デフォルト: 0, 範囲: 最小値 ～ 最大値)
        Track2 : パラメータ2 (デフォルト: 0, 範囲: 最小値 ～ 最大値)
        Track3 : パラメータ3 (デフォルト: 0, 範囲: 最小値 ～ 最大値)

        ※各トラックバーはアニメーションスライダーに対応しています(補間あり)。
        ※スクリプトエディタのツールバーから .lua ファイルのインポート/エクスポートが可能です。
        ※チェック(obj.check0〜obj.check3)とカラー(color)は、スクリプト本体でそれらを
          参照すると自動的にエフェクトパネルへ表示されます(トラックバーは常に表示)。
          この一般利用では範囲・既定値の宣言を伴わないため、値はスクリプト内で扱います。
          --track / --check / --color ヘッダによる宣言表示も従来どおり利用できます。

        数値スライダーが 4 つあり、各値は obj.slider0 ～ obj.slider3 として参照できます。
        値は整数で、小数は扱いません。

        Slider0 : 整数スライダー (範囲: 1 ～ 10)
        Slider1 : 整数スライダー (範囲: 0 ～ 100)
        Slider2 : 整数スライダー (範囲: 0 ～ 1000)
        Slider3 : 整数スライダー (範囲: -1000 ～ 1000)

        文字列パラメータも利用でき、各値は以下の名前で参照できます。

        obj.text          : テキスト入力欄の文字列
        obj.font          : フォント選択欄のフォント名
        obj.dir           : フォルダ選択欄のパス
        obj.file_video    : 動画ファイルのパス
        obj.file_audio    : 音声ファイルのパス
        obj.file_image    : 画像ファイルのパス
        obj.file_project  : プロジェクトファイルのパス
        obj.file_mp4      : MP4 ファイルのパス
        obj.file_exo      : EXO ファイルのパス
        obj.file_subtitle : 字幕ファイルのパス
        obj.file_shader   : シェーダーファイルのパス

        ※スライダーと文字列パラメータは、スクリプト本体でそれらを参照すると
          自動的にエフェクトパネルへ表示されます。
        ※ファイル選択のパスは obj.load へ渡して画像・動画の読み込みに使えます。
        ※これらのパラメータは MoonSharp とネイティブのどちらのエンジンでも参照できます。


- グローバル変数 -

        以下の変数はスクリプトのグローバルスコープで直接参照できます。

        time          : オブジェクト基準の現在時間(秒) [ReadOnly]
                        ※ obj.time と同じ値です。
        frame         : オブジェクト基準の現在フレーム番号 [ReadOnly]
                        ※ obj.frame と同じ値です。
        totalframe    : オブジェクトの総フレーム数 [ReadOnly]
                        ※ obj.totalframe と同じ値です。
        framerate     : フレームレート(fps) [ReadOnly]
                        ※ obj.framerate と同じ値です。
        timelineframe : タイムライン全体基準の現在フレーム番号 [ReadOnly]
                        ※ obj.frame はオブジェクト先頭からの相対フレームですが、
                          こちらはタイムライン先頭からの絶対フレームです。
        timelinetime  : タイムライン全体基準の現在時間(秒) [ReadOnly]
        layer         : オブジェクトが配置されているレイヤー番号 [ReadOnly]
                        ※ obj.layer と同じ値です。
        obj           : オブジェクト情報テーブル (下記「obj テーブル」を参照)
        scene         : シーン情報テーブル (下記「scene テーブル」を参照)
        ymm4          : YMM4固有のメタ情報テーブル (下記「ymm4 テーブル」を参照)
        anim          : アニメーション用ユーティリティテーブル (下記「anim テーブル」を参照)


- scene テーブル -

        scene.width   : シーン(プロジェクト)の横解像度(ピクセル) [ReadOnly]
        scene.height  : シーン(プロジェクト)の縦解像度(ピクセル) [ReadOnly]
        scene.cx      : シーンの横解像度の半分 (= scene.width / 2) [ReadOnly]
        scene.cy      : シーンの縦解像度の半分 (= scene.height / 2) [ReadOnly]


- obj テーブル - 変数 -

        以下の変数はオブジェクトの現在の描画情報を保持します。
        ※ ReadOnly と記載のないものはスクリプト内で変更可能であり、変更した値が描画に反映されます。

        ○座標・オフセット
        obj.x         : 表示基準座標 X (変更可)
                        ※ AviUtl2 仕様の「表示基準座標X」に相当します。
                        ※ 実装上は DrawDescription.Draw.X から設定され、
                          スクリプト後に変更内容が反映されます。
        obj.y         : 表示基準座標 Y (変更可)
                        ※ 実装上は DrawDescription.Draw.Y から設定されます。
        obj.z         : 表示基準座標 Z (変更可)
                        ※ 実装上は DrawDescription.Draw.Z から設定されます。
        obj.ox        : 中心オフセット X (変更可)
                        ※ 実装上は DrawDescription.CenterPoint.X (inDesc) から設定されます。
                        ※ AviUtl2 仕様の「基準座標からの相対座標X(obj.ox)」に相当しますが、
                          このプラグインでは CenterPoint(中心点のオフセット)として扱われます。
        obj.oy        : 中心オフセット Y (変更可)
                        ※ 実装上は DrawDescription.CenterPoint.Y (inDesc) から設定されます。

        ○拡大率・変形
        obj.zoom      : 拡大率(変更可)
                        ※ 0.0 が等倍相当ではなく、1.0 が等倍です(AviUtl2 互換)。
                        ※ 実装上は ZoomX と ZoomY の平均値 (zoomAvg) として算出されます。
                        ※ 出力時は obj.zoom * (1 ± obj.aspect) で X/Y 個別の Zoom に展開されます。
        obj.aspect    : アスペクト比(変更可) 範囲: -1.0 ～ 1.0
                        ※ 正の値で X 方向に縮小、負の値で Y 方向に縮小。
                        ※ 実装上は (ZoomX - ZoomY) / (ZoomX + ZoomY) として算出されます。
        obj.sx        : X 方向の拡大率 (変更可) 1.0 で等倍
                        ※ YMM4独自の精密な独立スケール制御です。
                        ※ zoom/aspect の変更よりも優先して反映されます。
        obj.sy        : Y 方向の拡大率 (変更可) 1.0 で等倍
                        ※ YMM4独自の精密な独立スケール制御です。
                        ※ zoom/aspect の変更よりも優先して反映されます。
        obj.alpha     : 不透明度(変更可) 範囲: 0.0 ～ 255.0
                        ※ AviUtl2 では 0.0～1.0 ですが、このプラグインでは 0.0～255.0 です。
                        ※ 実装上は DrawDescription.Opacity * 255 から設定されます。
                        ※ 出力時に 255 で除算し、0.0～1.0 に正規化されます(Clamp あり)。
                        ※ デフォルトスクリプトでは obj.alpha = math.min(time * 255, 255) が
                          設定されており、フェードインのデフォルト実装となっています。

        ○回転
        obj.rx        : X 軸回転角度(変更可) 単位: 度(°), 360.0 で一回転
                        ※ 実装上は DrawDescription.Rotation.X から設定されます。
        obj.ry        : Y 軸回転角度(変更可) 単位: 度(°), 360.0 で一回転
                        ※ 実装上は DrawDescription.Rotation.Y から設定されます。
        obj.rz        : Z 軸回転角度(変更可) 単位: 度(°), 360.0 で一回転
                        ※ 実装上は DrawDescription.Rotation.Z から設定されます。
        obj.rxr       : X 軸回転角度(ラジアン) (変更可) 単位: ラジアン(rad)
                        ※ rx と同時に変更された場合はこちらが優先して反映されます。
        obj.ryr       : Y 軸回転角度(ラジアン) (変更可) 単位: ラジアン(rad)
                        ※ ry と同時に変更された場合はこちらが優先して反映されます。
        obj.rzr       : Z 軸回転角度(ラジアン) (変更可) 単位: ラジアン(rad)
                        ※ rz と同時に変更された場合はこちらが優先して反映されます。

        ○画像サイズ
        obj.w         : 入力画像の横ピクセル数 [ReadOnly]
                        ※ 実装上は D2D の GetImageLocalBounds から算出した整数値です。
        obj.h         : 入力画像の縦ピクセル数 [ReadOnly]
                        ※ 実装上は D2D の GetImageLocalBounds から算出した整数値です。
        obj.hw        : 入力画像の横ピクセル数の半分 (= obj.w / 2) [ReadOnly]
        obj.hh        : 入力画像の縦ピクセル数の半分 (= obj.h / 2) [ReadOnly]
        obj.diagonal  : 画像の対角線の長さ (= math.sqrt(w*w + h*h)) [ReadOnly]
        obj.cx        : 画像の中心 X 座標 (= obj.w / 2) [ReadOnly]
                        ※ ピクセル操作における中心座標の参照値です。
        obj.cy        : 画像の中心 Y 座標 (= obj.h / 2) [ReadOnly]

        ○時間・フレーム
        obj.time      : オブジェクト基準の現在時間(秒) [ReadOnly]
                        ※ グローバル変数 time と同じ値です。
                        ※ 実装上は EffectDescription.ItemPosition.Time.TotalSeconds から設定されます。
        obj.frame     : オブジェクト基準の現在フレーム番号 [ReadOnly]
                        ※ グローバル変数 frame と同じ値です。
                        ※ 実装上は EffectDescription.ItemPosition.Frame から設定されます。
        obj.totalframe: オブジェクトの総フレーム数 [ReadOnly]
                        ※ グローバル変数 totalframe と同じ値です。
                        ※ 実装上は EffectDescription.ItemDuration.Frame から設定されます。
        obj.t         : オブジェクト内の現在時間の進行割合 (0.0 ～ 1.0) [ReadOnly]
        obj.framerate : フレームレート(fps) [ReadOnly]
                        ※ グローバル変数 framerate と同じ値です。
                        ※ 実装上は EffectDescription.FPS から設定されます。

        ○トラックバー
        obj.track0    : エフェクトパネルの Track0 の現在値 [ReadOnly]
                        ※ アニメーション補間後の値です。
        obj.track1    : エフェクトパネルの Track1 の現在値 [ReadOnly]
        obj.track2    : エフェクトパネルの Track2 の現在値 [ReadOnly]
        obj.track3    : エフェクトパネルの Track3 の現在値 [ReadOnly]

        ○スライダー
        obj.slider0   : エフェクトパネルの Slider0 の現在値(整数) [ReadOnly]
        obj.slider1   : エフェクトパネルの Slider1 の現在値(整数) [ReadOnly]
        obj.slider2   : エフェクトパネルの Slider2 の現在値(整数) [ReadOnly]
        obj.slider3   : エフェクトパネルの Slider3 の現在値(整数) [ReadOnly]

        ○文字列パラメータ
        obj.text      : テキスト入力欄の文字列 [ReadOnly]
        obj.font      : フォント選択欄のフォント名 [ReadOnly]
        obj.dir       : フォルダ選択欄のパス [ReadOnly]
        obj.file_video 〜 obj.file_shader : 各ファイル選択欄のパス [ReadOnly]

        ○レイヤー
        obj.layer     : オブジェクトが配置されているレイヤー番号 [ReadOnly]
                        ※ グローバル変数 layer と同じ値です。
                        ※ 実装上は EffectDescription.Layer から設定されます。


- obj テーブル - 変数の変更と描画への反映 -

        スクリプト実行後、以下の変数のみが描画情報に読み戻されます(反映されます)。

        反映される変数:
            obj.x, obj.y, obj.z
            obj.ox, obj.oy
            obj.zoom, obj.aspect
            obj.alpha
            obj.rx, obj.ry, obj.rz

        ※ obj.w, obj.h, obj.cx, obj.cy, obj.frame, obj.time,
           obj.totalframe, obj.framerate, obj.track0 ～ obj.track3,
           obj.layer は ReadOnly であり、変更しても描画には反映されません。

        出力値の算出式:
            ZoomX = obj.zoom * (1.0 + obj.aspect)
            ZoomY = obj.zoom * (1.0 - obj.aspect)
            Opacity = Clamp(obj.alpha / 255.0, 0.0, 1.0)
            Draw = (obj.x, obj.y, obj.z)
            CenterPoint = (obj.ox, obj.oy)
            Rotation = (obj.rx, obj.ry, obj.rz)


- obj テーブル - 関数 -

        ○obj.getobject(tag [, frame])
                指定したタグを持つ他のオブジェクトの描画情報を取得します。
                ※ タグは対象アイテムの「備考」に入力した文字列と一致します。
                ※ 第2引数 frame を指定すると、そのタイムラインフレーム時点の値を
                  取得します。省略時は現在フレームの値を取得します。
                ※ 同じタグを持つアイテムが複数ある場合、対象フレームで表示中の
                  アイテムを優先して返します。表示中のものが無い場合は最初に
                  一致したアイテムを返します。
                ※ 取得できる値はアイテム自身の座標・回転・拡大率です。
                  グループ制御・カメラ制御・上位エフェクトによる変形は含まれません。
                ※ 各値は obj テーブルと同一の単位系です。
                tag     : 対象アイテムの備考に設定した文字列
                frame   : 取得するタイムライン上の絶対フレーム (省略可)
                戻り値  : オブジェクト情報テーブル。一致するアイテムが存在しない場合は
                          nil を返します。
                          .exist : 対象フレームで表示中かどうか (boolean)
                          .x     : 表示基準座標 X
                          .y     : 表示基準座標 Y
                          .z     : 表示基準座標 Z
                          .zoom  : 拡大率 (1.0 で等倍)
                          .sx    : X 方向の拡大率 (= zoom)
                          .sy    : Y 方向の拡大率 (= zoom)
                          .rx    : X 軸回転角度 (度, 常に 0)
                          .ry    : Y 軸回転角度 (度, 常に 0)
                          .rz    : Z 軸回転角度 (度)
                          .rxr   : X 軸回転角度 (ラジアン, 常に 0)
                          .ryr   : Y 軸回転角度 (ラジアン, 常に 0)
                          .rzr   : Z 軸回転角度 (ラジアン)
                          .alpha : 不透明度 (0.0 ～ 255.0, フェード反映済み)
                          .layer : 配置レイヤー番号
                例：
                    local funnel = obj.getobject("funnel")
                    if funnel then
                        obj.x = funnel.x
                        obj.y = funnel.y
                    end
                発射時の座標に固定する例 (発射元の備考が "funnel" の場合)：
                    local fire = timelineframe - frame
                    local src = obj.getobject("funnel", fire)
                    if src then
                        obj.x = src.x
                        obj.y = src.y
                    end

        ○obj.getpixel(x, y)
                指定座標のピクセル情報を取得します。
                ※ 初回呼び出し時に GPU→CPU への画像データ転送が発生します(処理負荷を伴います)。
                ※ ピクセル座標は画像の左上を (0, 0) とし、右・下方向が正です。
                ※ 取得値はプリマルチプライドアルファを逆変換したストレートアルファ相当の値です。
                x       : X 座標 (0 ～ obj.w - 1)
                y       : Y 座標 (0 ～ obj.h - 1)
                戻り値  : r, g, b, a の 4 値(多値返却)
                          r = 赤成分 (0.0 ～ 255.0)
                          g = 緑成分 (0.0 ～ 255.0)
                          b = 青成分 (0.0 ～ 255.0)
                          a = アルファ値 (0.0 ～ 255.0)
                          ※ 範囲外の座標を指定した場合は (0, 0, 0, 0) を返します。
                例：r, g, b, a = obj.getpixel(10, 20)

        ○obj.setpixel(x, y, r, g, b [, a])
                指定座標のピクセル情報を書き換えます。
                ※ 呼び出し時に自動的にピクセルデータが「変更済み」としてマークされ、
                  スクリプト終了後に変更後の画像が出力されます。
                ※ 内部ではプリマルチプライドアルファ形式(BGRA)で保持されます。
                   setpixel 時に premultiplied 変換が行われます。
                x       : X 座標 (0 ～ obj.w - 1)
                y       : Y 座標 (0 ～ obj.h - 1)
                r       : 赤成分 (0.0 ～ 255.0)
                g       : 緑成分 (0.0 ～ 255.0)
                b       : 青成分 (0.0 ～ 255.0)
                a       : アルファ値 (省略時は 255.0)
                例：obj.setpixel(10, 20, 255, 0, 0, 128)

        ○obj.getpixeldata()
                全ピクセルデータに一括アクセスするためのプロキシオブジェクトを返します。
                ※ getpixel/setpixel と比較して高速にピクセルを操作できます。
                ※ 初回呼び出し時に GPU→CPU への画像データ転送が発生します(処理負荷を伴います)。
                戻り値  : PixelDataProxy オブジェクト
                          .width  : 画像の横ピクセル数 (= obj.w)
                          .height : 画像の縦ピクセル数 (= obj.h)
                          :get(index)  : インデックス指定でチャンネル値を取得
                          :set(index, value) : インデックス指定でチャンネル値を設定

                インデックスの仕様:
                    インデックスは 1 始まりです(Lua の慣習に準拠)。
                    チャンネルの順序は R, G, B, A の繰り返しです。
                    pixel(x, y) の R チャンネルのインデックス =
                        (y * obj.w + x) * 4 + 1
                    pixel(x, y) の G チャンネルのインデックス =
                        (y * obj.w + x) * 4 + 2
                    pixel(x, y) の B チャンネルのインデックス =
                        (y * obj.w + x) * 4 + 3
                    pixel(x, y) の A チャンネルのインデックス =
                        (y * obj.w + x) * 4 + 4

                    ※ アルファ(チャンネル4)を変更した場合は RGB も比率変換されます。
                    ※ RGB チャンネルを変更した場合は premultiplied 変換が行われます。
                    ※ 取得値は get(index) 同様ストレートアルファ相当です。

                例：
                    local pd = obj.getpixeldata()
                    local idx = (10 * obj.w + 20) * 4 + 1  -- (x=20, y=10) の R チャンネル
                    local r = pd:get(idx)
                    pd:set(idx, 128)

        ○obj.putpixeldata()
                互換性のために定義された空実装関数です(no-op)。
                ※ AviUtl のスクリプトでは getpixeldata/putpixeldata の組み合わせで使用しますが、
                  このプラグインでは setpixel/set による変更が自動的に反映されるため、
                  本関数を呼び出す必要はありません。

        ○obj.rand(st, ed [, seed [, frame]])
                指定範囲 st ～ ed の整数を返す決定論的な乱数です。
                ※ seed と frame が同じであれば常に同じ値を返します。
                ※ seed 省略時は 0、frame 省略時は現在のフレーム(obj.frame)を使用します。
                ※ 従来エンジン(MoonSharp)と高速ランタイム(LuaJIT)で同じ値を返します。
                  エンジンに依存しない小数の乱数が必要な場合は anim.rand を使用してください。
                st      : 範囲の下限 (小数は切り捨てて整数化)
                ed      : 範囲の上限 (小数は切り捨てて整数化)
                seed    : 乱数シード (省略可, デフォルト 0)
                frame   : 変化に用いるフレーム (省略可, デフォルト obj.frame)
                戻り値  : st ～ ed の整数
                例：obj.ox = obj.ox + obj.rand(-10, 10, 1, obj.frame)

        ○obj.load(対象, ...)
                オブジェクトのバッファを生成した内容に置き換えます。
                対象は "figure"(図形) / "text"(テキスト) / "image"(画像ファイル) /
                "movie"(動画ファイル) に対応します。生成後 obj.w / obj.h 等は生成
                サイズへ更新され、load 直後の getpixel でも参照できます。
                従来エンジン・高速エンジンの両方で動作します。

                obj.load("figure", 名前, 色, サイズ [, ライン [, アスペクト]])
                        図形を生成してバッファを置き換えます(中心に配置)。
                        名前      : 円 / 四角形 / 三角形 / 五角形 / 六角形 / 星形
                                    (英語名 circle, rect, triangle, pentagon,
                                    hexagon, star も使用可)。
                        色        : 塗り色 0xRRGGBB。
                        サイズ    : 図形の基準サイズ(ピクセル)。
                        ライン    : 輪郭の太さ(省略可)。0 で塗りつぶし。
                        アスペクト: 歪曲 -1.0 ～ 1.0(省略可)。
                        例：obj.load("figure", "円", 0xff3366, 200)

                obj.load("text", 文字列)
                        obj.setfont で指定したフォント・サイズ・色で文字列を
                        描画してバッファを置き換えます(DirectWrite で描画)。
                        例：obj.setfont("Yu Gothic UI", 48); obj.load("text", "字幕")

                obj.load("image", パス)
                        画像ファイルを読み込んでバッファを置き換えます。
                        ※ 安全のため先頭ヘッダーを検証し、真正な画像
                          (PNG / JPEG / GIF / BMP / TIFF / WebP)のみ読み込みます。
                        例：obj.load("image", "C:\\img\\logo.png")

                obj.load("movie", パス [, 時間])
                        動画ファイルの指定時間のフレームを読み込んでバッファを
                        置き換えます(Media Foundation で復号)。
                        時間 : 秒。省略時は obj.time。
                        ※ 安全のため先頭ヘッダーを検証し、真正な動画
                          (MP4 / MOV / MKV / WebM / AVI / FLV / ASF)のみ
                          読み込みます。
                        例：obj.load("movie", "C:\\mov\\clip.mp4", obj.time)

        ○obj.setfont(フォント名 [, サイズ [, スタイル [, 色]]])
                obj.load("text") で使用するフォントを設定します。
                スタイル : ビット和。1=太字, 2=斜体, 3=太字+斜体。
                色       : 0xRRGGBB。
                例：obj.setfont("Meiryo", 64, 1, 0xffffff)

        ○obj.draw([ox, oy, oz [, 拡大率 [, 不透明度 [, アスペクト]]]])
                現在のオブジェクト画像を蓄積バッファへ合成して描画します。
                複数回呼ぶと重ねて描画されます(残像・パーティクル等)。
                ox, oy, oz : 中心からのオフセット(省略時 0)。
                拡大率     : 1.0 で等倍(省略時 1.0)。
                不透明度   : 0.0 ～ 1.0(省略時 1.0)。
                アスペクト : -1.0 ～ 1.0(省略時 0)。
                ※ 呼び出すと標準描画は抑制され、合成結果が出力されます。

        ○obj.drawpoly(x0,y0,z0, x1,y1,z1, x2,y2,z2, x3,y3,z3
                       [, u0,v0, u1,v1, u2,v2, u3,v3 [, 不透明度]])
                4 頂点で指定した四角形へオブジェクト画像をテクスチャ描画します。
                頂点は左上→右上→右下→左下の順。UV 省略時は画像全体を使用します。
                ※ UV を省略し 13 番目の引数のみ与えた場合、その値を不透明度として
                  使用します(座標 12 個 + 不透明度)。
                ※ アフィン変換で写像します(平行四辺形は厳密、非平行四辺形は近似)。

        ○obj.copybuffer(コピー先, コピー元)
                バッファ間で画像をコピーします。識別子の先頭文字で判定します。
                  "obj"        : オブジェクト画像バッファ
                  "tmp"        : 一時バッファ
                  "cache:名前" : 名前付きキャッシュ(フレーム間で保持)
                例：obj.copybuffer("tmp", "obj")  -- 退避
                    obj.copybuffer("obj", "tmp")  -- 復元
                ※ "frm"(フレームバッファ/背景)はYMM4の構造上取得できません。

        ○obj.getvalue(対象)
                obj の現在値を取得します(obj.x や obj.track0 等と同値)。
                対象   : "x","y","z","ox","oy","rx","ry","rz","zoom","aspect",
                         "alpha","sx","sy","track0"～"track3","time","frame" 等。
                戻り値 : 数値(未知の対象は 0)。

        ○obj.setoption(名前, 値) / obj.getoption(名前)
                オプションを設定 / 取得します。未対応の項目も受理してスクリプトを
                停止させません(getoption は設定値を返し、未設定は 0)。
                現在描画へ反映される項目：
                  "antialias" : 0 で最近傍、それ以外で線形補間(obj.draw/drawpoly)。
                  "blend" : obj.draw / obj.drawpoly の合成モードを番号で指定します。
                    YMM4 のプロパティパネルにある「合成モード」のドロップダウンと
                    並び順が同じで、番号は 0 から始まります。
                    0=通常,1=ディザ合成,2=比較（暗）,3=乗算,4=焼き込みカラー,
                    5=焼き込み（リニア）,6=比較（明）,7=スクリーン,8=覆い焼きカラー,
                    9=覆い焼き（リニア） - 加算,10=加算,11=オーバーレイ,
                    12=ソフトライト,13=ハードライト,14=ビビッドライト,15=リニアライト,
                    16=ピンライト,17=ハードミックス,18=差分,19=除外,20=減算,21=除算,
                    22=色相,23=彩度,24=カラー,25=輝度,26=カラー比較（明）,27=背景,
                    28=カラー比較（暗）,29=削除,30=背景でクリッピング,
                    31=重ならない部分のみ,32=色反転マスク。
                    範囲外の値や数値以外を指定した場合は通常モードになります。
                  "draw_state" : このエフェクトが作業画像を出力するかどうかを
                    真偽値で決めます。true で作業画像を出力し、false で入力を
                    そのまま通します。obj.draw や setpixel を使わずに作業画像を
                    そのまま出力させたいときや、逆に出力を止めたいときに使います。
                    両エンジンで動きます。
                  "drawtarget" : obj.draw / obj.drawpoly の描画先を切り替えます。
                    "framebuffer"(既定)ではフレームバッファへ合成します。
                    "tempbuffer" では作業用の tmp バッファへその場で合成します。
                    obj.setoption("drawtarget", "tempbuffer", w, h) の w, h は
                    tmp バッファを初期化するサイズで、省略すると初期化しません。
                    tmp へ描いた結果は obj.copybuffer("obj", "tmp") で作業画像へ
                    戻せます。tempbuffer への合成は通常(ソースオーバー)で、blend は
                    framebuffer への描画に適用されます。両エンジンで動きます。
                    MoonSharp では tempbuffer への合成を GPU で実行し、GPU を
                    利用できない環境では CPU の合成へ自動的に切り替わります。
                    丸めの違いにより、合成の結果はエンジン間でごく僅かに
                    異なることがあります。

        ○obj.pixeloption(名前, 値)
                ピクセル操作のオプションを状態として保持します。

        ○obj.setanchor(名前, 個数 [, オプション, ..])
                アンカーポイントをプレビュー上に表示し、マウスドラッグで座標を
                調整できるようにします。調整した座標はプロジェクトへ保存されます。
                名前 : 座標を格納する Lua 変数の名前を文字列で指定します。
                    変数は pos={x0,y0,x1,y1,…} の並びで、xyz 指定時は
                    {x0,y0,z0,x1,y1,z1,…} になります。座標はオブジェクトの
                    中心が原点です。
                個数 : アンカーポイントの数を指定します。最大 32 個です。
                オプション : 文字列で列挙できます。
                    "line" : アンカーポイントを線で結びます。
                    "loop" : アンカーポイントを線で結び一周させます。
                    "star" : 各アンカーポイントとオブジェクトの中心を線で結びます。
                    "arm"  : オブジェクトの中心から各アンカーポイントへ順に線で結びます。
                    "xyz"  : アンカーポイントを 3D 座標で扱います。既定は 2D です。
                戻り値は設置したアンカーポイントの数です。両エンジンで動きます。
                名前に "track" を指定するトラックバー連携は現在未対応で、0 を返します。
                例：
                  num = obj.track0
                  obj.setanchor("pos", num, "loop")
                  for i = 0, num - 1 do
                    obj.draw(pos[i * 2 + 1], pos[i * 2 + 2])
                  end

- スクリプト例 -

        ○フェードイン(デフォルトスクリプト)
                obj.alpha = math.min(time * 255, 255)

        ○フェードアウト
                obj.alpha = math.max((obj.totalframe - obj.frame) / obj.framerate * 255, 0)

        ○Track0 で透明度を制御
                obj.alpha = obj.track0

        ○時間に応じて回転
                obj.rz = time * 90  -- 1秒で90度回転

        ○ピクセルの赤成分を反転する
                for y = 0, obj.h - 1 do
                    for x = 0, obj.w - 1 do
                        local r, g, b, a = obj.getpixel(x, y)
                        obj.setpixel(x, y, 255 - r, g, b, a)
                    end
                end

        ○getpixeldata を使ったグレースケール変換(高速)
                local pd = obj.getpixeldata()
                local w = pd.width
                local h = pd.height
                for y = 0, h - 1 do
                    for x = 0, w - 1 do
                        local base = (y * w + x) * 4
                        local r = pd:get(base + 1)
                        local g = pd:get(base + 2)
                        local b = pd:get(base + 3)
                        local a = pd:get(base + 4)
                        local gray = r * 0.299 + g * 0.587 + b * 0.114
                        pd:set(base + 1, gray)
                        pd:set(base + 2, gray)
                        pd:set(base + 3, gray)
                    end
                end

        ○Track0 でズームを制御 (1.0 = 等倍)
                obj.zoom = obj.zoom * (1 + obj.track0 / 100)


- AviUtl スクリプトの部分互換 -

        AviUtl のアニメーション効果スクリプト(*.anm 系)のうち、座標・拡縮・回転・
        不透明度の変形と乱数を用いる範囲を、追加の記述変更なしで実行できる互換層です。
        スクリプトに AviUtl 固有の記法が含まれない場合、この互換層は一切介入しません
        (従来のスクリプトの挙動は変わりません)。

        ○スクリプトヘッダの解釈
                コメント形式のパラメータ宣言を解釈し、スクリプト本体が参照する変数を
                実行前に定義します。
                --dialog:項目名,local 変数=既定値;…
                        各項目の宣言部(local 変数=既定値 など)を本体の先頭へ展開します。
                        AviUtl では対話ダイアログで編集する値ですが、ここでは既定値で
                        動作します。
                --color:0xRRGGBB
                        変数 color を指定色で定義します。
                --check0:項目名,既定値
                        obj.check0 (0～3) を既定値で定義します。
                --param:任意のLua文
                        記述した Lua 文をそのまま本体の先頭へ展開します。
                --track0: ～ --track3:
                        トラックバーの値は従来どおり obj.track0 ～ obj.track3 から
                        参照します。ヘッダーのラベル・範囲・既定値は UI へ反映しません。

        ○@ による複数セクション
                1 つのスクリプトに @名前 で複数の効果が定義されている場合、先頭の
                セクションのみを実行します。

        ○obj.rand(st, ed [, seed [, frame]])
                AviUtl 互換の決定論的な整数乱数です。詳細は「obj テーブル - 関数」を
                参照してください。

        ○描画・合成・読み込み
                obj.draw / obj.drawpoly による描画合成、obj.copybuffer による
                バッファ操作、obj.load("figure"/"text"/"image"/"movie") による生成・
                読み込み、obj.setfont、obj.effect、obj.getvalue、obj.setoption /
                obj.getoption / obj.pixeloption に対応します。詳細は「obj テーブル -
                関数」を参照してください。

        ○obj.effect の AviUtl／AviUtl2 互換マップ
                obj.effect("エフェクト名", "パラメータ名", 値, …) はまず互換マップを
                参照します。AviUtl／AviUtl2 の名前として宣言したエフェクトは、値がマップに
                定義した倍率とオフセットの変換を経て YMM4 のプロパティへ反映します。互換
                マップに宣言が無い名前は、YMM4 のエフェクト名やキーワードでそのまま解決
                するため、既存の呼び出しはそのまま動きます。この互換マップは、YMM4 自身が AviUtl の
                .exo へ書き出す変換を逆向きにたどって作っているため、往復しても結果が
                一致します。対象のエンジンは指定子 --!aviutl または --!aviutl2 で
                切り替えます。指定子を書かないときは --!aviutl として扱います。
                マップに宣言したエフェクト名と主なパラメータは次のとおりです。
                  ぼかし: 範囲, サイズ固定
                  境界ぼかし: 範囲
                  方向ブラー: 範囲, 角度, サイズ固定
                  放射ブラー: 範囲, X, Y, サイズを固定
                  閃光: 範囲, X, Y, サイズを固定
                  クロマキー: 色相範囲, 境界補正
                  クリッピング: 上, 下, 左, 右, 中心の位置を変更
                  斜めクリッピング: 中心X, 中心Y, 角度, ぼかし, 幅
                  領域拡張: 上, 下, 左, 右
                  座標: X, Y, Z
                  回転: X, Y, Z
                  リサイズ: 拡大率, X, Y, 補間なし
                  画像ループ: 横回数, 縦回数
                  極座標変換: 中心幅, 渦巻
                  グラデーション: 強さ, 中心X, 中心Y, 角度, 幅
                  単色化: 強さ, 輝度を保持する
                  縁取り: サイズ, ぼかし
                  反転: パラメータなし
                  モザイク: サイズ
                  ディスプレイスメントマップ: ぼかし
                  動画ファイル合成: X, Y, 拡大率
                  色調補正: 明るさ, ｺﾝﾄﾗｽﾄ, 色相, 輝度, 彩度
                  色ずれ: ずれ幅, 角度, 強さ
                色調補正 と 色ずれ はコミュニティプラグインのエフェクトに対応します。

        次の機能はこの互換層では対象外です。
        これらを呼び出すスクリプトは意図どおりには動作しません。
                ×require によるモジュール読み込み、io / dofile などの任意ファイル
                  アクセス(obj.load の画像 / 動画はヘッダー検証付きで対応)。
                ×obj.setanchor、scene.set / scene.get 等。


- 非対応・制限事項 -

        ×AviUtl2 の obj.rand1(), obj.mes(), obj.setanchor() 等には対応していません。
        ×AviUtl2 の scene テーブルの scene.set(), scene.get() 等の関数には対応していません。
          (scene.width と scene.height の読み取りのみ対応)
        ×スクリプトからの任意ファイルアクセス(require, dofile, io 等)には対応して
          いません。obj.load("image"/"movie") のみ、先頭ヘッダー検証で真正なメディアと
          確認できたファイルを読み込みます。
        ×obj.z を変更した場合、描画には反映されます(ReadBack に含まれる)が、
          3D 描画のサポートは限定的です。
        ×スクリプトのタイムアウト(5秒)を超えた場合、実行は強制終了されます。
          無限ループが発生しないよう実装上の注意が必要です。
        ×AviUtl2 のヘッダーディレクティブは部分的にのみ対応します。
          --dialog / --color / --check / --param の既定値は解釈されますが
          (「AviUtl スクリプトの部分互換」を参照)、--track@ のラベル・範囲・
          既定値は UI へ反映されず、値は Track0 ～ Track3 を使用します。
