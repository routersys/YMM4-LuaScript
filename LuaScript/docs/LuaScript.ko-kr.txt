- Lua 스크립트 이펙트에 대하여 -

        YukkuriMovieMaker4의 「Lua 스크립트」 영상 이펙트에서
        사용 가능한 스크립트에는 Lua 언어를 사용합니다.
        스크립트 엔진으로는 MoonSharp (Lua 5.2 호환)이 채용되어 있습니다.

        ※ 이 이펙트는 AviUtl2의 스크립트 사양과는 다른 독자적인 구현입니다.
        ※ AviUtl2의 스크립트 파일(*.anm2 등)과는 별도의 사양입니다.
        ※ IsAviUtlSupported = false이므로 AviUtl용 EXO 출력은 지원하지 않습니다.


- 주의 사항 -

        ○ 스크립트 실행 엔진은 MoonSharp (Lua 5.2 호환)입니다.
        ○ 사용 가능한 표준 라이브러리는 다음으로 제한됩니다.
            table, string, math, bit32
            ※ io, os, debug, ffi, require 등은 사용할 수 없습니다.
        ○ 스크립트 타임아웃은 5000ms (5초)입니다.
            타임아웃 시 실행이 강제 종료되며 오류로 처리됩니다.
        ○ 스크립트에서 파일 시스템에 접근할 수 없습니다 (require 포함).
        ○ 스크립트 내부에서 외부 스크립트를 호출하는 것은 지원하지 않습니다.
        ○ 픽셀 조작(getpixel/setpixel/getpixeldata)은 실제 입력 이미지에
            접근하므로 최초 접근 시에만 GPU→CPU로의 데이터 전송이
            발생합니다. 필요 없는 경우 호출을 생략하면 처리를 빠르게 할 수 있습니다.


- 실행 엔진과 자동 가속 -

        픽셀 처리(getpixel/setpixel 이중 루프)를 포함하는 스크립트는
        구문을 분석하여 자동으로 더 빠른 엔진으로 분배됩니다.
        좌표・회전・불투명도 등 일반 스크립트는 기존과 동일하게 MoonSharp으로
        실행되며 동작은 전혀 바뀌지 않습니다.

        ○ 자동 분배(디렉티브가 없을 때)
                · 자기 픽셀로 완결되는 픽셀 매핑(아래 조건) → CPU 커널(고속・엄밀)
                · 위 이외의 픽셀 조작 → 네이티브(LuaJIT, 배치된 경우에만)
                · 픽셀 조작 없음 → MoonSharp

        ○ 엔진 지정 디렉티브(스크립트 첫 줄의 주석)
                --!cpu       : CPU 커널. 식 트리로 런타임 컴파일하여
                               모든 CPU 코어에서 병렬 실행합니다(MoonSharp과 완전 일치).
                --!gpu       : GPU 커널. Direct2D 픽셀 셰이더로 실행하여
                               GPU→CPU 전송을 없애 가장 빠릅니다.
                               float32 정밀도이므로 결과가 아주 약간 다를 수 있습니다(옵트인).
                --!native    : LuaJIT(별도 프로세스). 고속이며 MoonSharp과 일치.
                               native 폴더의 배치가 필요합니다.
                --!moonsharp : 기존의 MoonSharp을 강제합니다.
                ※ 지정이 사용 불가/미지원인 경우 자동으로 하위 엔진으로
                  폴백합니다(출력이 깨지지 않습니다).

        ○ 고속 커널이 성립하는 구문(순수 픽셀 매핑)
                다음 형태의 이중 루프만 커널화됩니다.
                    for y = 0, obj.h - 1 do
                        for x = 0, obj.w - 1 do
                            local r, g, b, a = obj.getpixel(x, y)
                            obj.setpixel(x, y, r2, g2, b2 [, a2])
                        end
                    end
                · 본문에서 사용할 수 있는 것은 사칙연산・math.*・비교・임계값
                  「조건 and 값1 or 값2」・local 변수뿐입니다.
                · 참조할 수 있는 것은 처리 중인 자기 픽셀(getpixel(x, y))과
                  obj/scene/time 등의 값뿐입니다.
                · 이웃 참조(블러 등), 다른 좌표로의 쓰기, getpixeldata,
                  getobject, 테이블/문자열 조작, if 문 등을 포함하는 경우에는
                  커널화되지 않고 자동으로 기존 엔진에서 실행됩니다.

        예(흰색 도형이 빨갛게 됨):
                --!gpu
                for y = 0, obj.h - 1 do
                    for x = 0, obj.w - 1 do
                        local r, g, b, a = obj.getpixel(x, y)
                        obj.setpixel(x, y, r, g / 2, b / 2, a)
                    end
                end


- 파라미터 -

        이펙트 패널에는 다음 4개의 트랙바(애니메이션 지원)가 있습니다.
        각 값은 스크립트 내에서 obj.track0 ～ obj.track3으로 참조할 수 있습니다.

        Track0 : 파라미터 0 (기본값: 0, 범위: 최솟값 ～ 최댓값)
        Track1 : 파라미터 1 (기본값: 0, 범위: 최솟값 ～ 최댓값)
        Track2 : 파라미터 2 (기본값: 0, 범위: 최솟값 ～ 최댓값)
        Track3 : 파라미터 3 (기본값: 0, 범위: 최솟값 ～ 최댓값)

        ※ 각 트랙바는 애니메이션 슬라이더를 지원합니다(보간 있음).
        ※ 스크립트 편집기의 툴바에서 .lua 파일을 가져오기/내보내기할 수 있습니다.
        ※ 체크박스(obj.check0〜obj.check3)와 색상(color)은 스크립트 본문에서 참조하면
          이펙트 패널에 자동으로 표시됩니다(트랙바는 항상 표시). 이 일반 사용은 범위・기본값
          선언을 동반하지 않으므로 값의 범위는 스크립트 내에서 처리하세요. AviUtl의
          --track / --check / --color 헤더 선언 표시도 기존과 동일하게 사용할 수 있습니다.

        숫자 슬라이더가 4개 있으며, 각 값은 obj.slider0 ～ obj.slider3으로 참조할 수 있습니다.
        값은 정수이며 소수는 다루지 않습니다.

        Slider0 : 정수 슬라이더 (범위: 1 ～ 10)
        Slider1 : 정수 슬라이더 (범위: 0 ～ 100)
        Slider2 : 정수 슬라이더 (범위: 0 ～ 1000)
        Slider3 : 정수 슬라이더 (범위: -1000 ～ 1000)

        문자열 파라미터도 사용할 수 있으며, 각 값은 다음 이름으로 참조합니다.

        obj.text          : 텍스트 입력란의 문자열
        obj.font          : 글꼴 선택란의 글꼴 이름
        obj.dir           : 폴더 선택란의 경로
        obj.file_video    : 동영상 파일의 경로
        obj.file_audio    : 음성 파일의 경로
        obj.file_image    : 이미지 파일의 경로
        obj.file_project  : 프로젝트 파일의 경로
        obj.file_mp4      : MP4 파일의 경로
        obj.file_exo      : EXO 파일의 경로
        obj.file_subtitle : 자막 파일의 경로
        obj.file_shader   : 셰이더 파일의 경로

        ※슬라이더와 문자열 파라미터는 스크립트 본문에서 참조하면 이펙트 패널에 자동으로 표시됩니다.
        ※파일 선택 경로는 obj.load에 전달해 이미지・동영상을 불러오는 데 사용할 수 있습니다.
        ※이 파라미터들은 MoonSharp과 네이티브 두 엔진에서 모두 참조할 수 있습니다.


- 전역 변수 -

        다음 변수는 스크립트의 전역 범위에서 직접 참조할 수 있습니다.

        time          : 오브젝트 기준 현재 시간(초) [읽기 전용]
                        ※ obj.time과 동일한 값입니다.
        frame         : 오브젝트 기준 현재 프레임 번호 [읽기 전용]
                        ※ obj.frame과 동일한 값입니다.
        totalframe    : 오브젝트의 총 프레임 수 [읽기 전용]
                        ※ obj.totalframe과 동일한 값입니다.
        framerate     : 프레임 레이트(fps) [읽기 전용]
                        ※ obj.framerate와 동일한 값입니다.
        timelineframe : 타임라인 전체 기준 현재 프레임 번호 [읽기 전용]
                        ※ obj.frame은 오브젝트 시작 지점으로부터의 상대 프레임이지만,
                          이 값은 타임라인 시작 지점으로부터의 절대 프레임입니다.
        timelinetime  : 타임라인 전체 기준 현재 시간(초) [읽기 전용]
        layer         : 오브젝트가 배치된 레이어 번호 [읽기 전용]
                        ※ obj.layer와 동일한 값입니다.
        obj           : 오브젝트 정보 테이블 (아래「obj 테이블」참조)
        scene         : 씬 정보 테이블 (아래「scene 테이블」참조)
        ymm4          : YMM4 고유 메타 정보 테이블 (아래 'ymm4 테이블' 참조)
        anim          : 애니메이션용 유틸리티 테이블 (아래 'anim 테이블' 참조)


- scene 테이블 -

        scene.width   : 씬(프로젝트)의 가로 해상도(픽셀) [읽기 전용]
        scene.height  : 씬(프로젝트)의 세로 해상도(픽셀) [읽기 전용]


- obj 테이블 - 변수 -

        다음 변수는 오브젝트의 현재 렌더링 정보를 보유합니다.
        ※ 읽기 전용으로 표시되지 않은 것은 스크립트 내에서 변경 가능하며, 변경된 값이 렌더링에 반영됩니다.

        ○ 좌표·오프셋
        obj.x         : 표시 기준 좌표 X (변경 가능)
                        ※ AviUtl2 사양의「표시 기준 좌표 X」에 해당합니다.
                        ※ 구현 상 DrawDescription.Draw.X에서 설정되며,
                          스크립트 실행 후 변경 내용이 반영됩니다.
        obj.y         : 표시 기준 좌표 Y (변경 가능)
                        ※ 구현 상 DrawDescription.Draw.Y에서 설정됩니다.
        obj.z         : 표시 기준 좌표 Z (변경 가능)
                        ※ 구현 상 DrawDescription.Draw.Z에서 설정됩니다.
        obj.ox        : 중심 오프셋 X (변경 가능)
                        ※ 구현 상 DrawDescription.CenterPoint.X (inDesc)에서 설정됩니다.
                        ※ AviUtl2 사양의「기준 좌표로부터의 상대 좌표 X(obj.ox)」에 해당하지만,
                          이 플러그인에서는 CenterPoint(중심점 오프셋)로 처리됩니다.
        obj.oy        : 중심 오프셋 Y (변경 가능)
                        ※ 구현 상 DrawDescription.CenterPoint.Y (inDesc)에서 설정됩니다.

        ○ 확대율·변형
        obj.zoom      : 확대율(변경 가능)
                        ※ 0.0이 등배가 아니라 1.0이 등배입니다(AviUtl2 호환).
                        ※ 구현 상 ZoomX와 ZoomY의 평균값(zoomAvg)으로 산출됩니다.
                        ※ 출력 시 obj.zoom * (1 ± obj.aspect)로 X/Y 각각의 Zoom으로 전개됩니다.
        obj.aspect    : 종횡비(변경 가능) 범위: -1.0 ～ 1.0
                        ※ 양수 값은 X 방향으로 축소, 음수 값은 Y 방향으로 축소.
                        ※ 구현 상 (ZoomX - ZoomY) / (ZoomX + ZoomY)로 산출됩니다.
        obj.sx        : X 방향 확대율 (변경 가능) 1.0이 100%
                        ※ YMM4 고유의 정밀한 독립 스케일 제어입니다.
                        ※ zoom/aspect 변경보다 우선하여 반영됩니다.
        obj.sy        : Y 방향 확대율 (변경 가능) 1.0이 100%
                        ※ YMM4 고유의 정밀한 독립 스케일 제어입니다.
                        ※ zoom/aspect 변경보다 우선하여 반영됩니다.
        obj.alpha     : 불투명도(변경 가능) 범위: 0.0 ～ 255.0
                        ※ AviUtl2에서는 0.0～1.0이지만, 이 플러그인에서는 0.0～255.0입니다.
                        ※ 구현 상 DrawDescription.Opacity * 255에서 설정됩니다.
                        ※ 출력 시 255로 나누어 0.0～1.0으로 정규화됩니다(Clamp 있음).
                        ※ 기본 스크립트에서는 obj.alpha = math.min(time * 255, 255)가
                          설정되어 있으며, 페이드인의 기본 구현으로 되어 있습니다.

        ○ 회전
        obj.rx        : X 축 회전 각도(변경 가능) 단위: 도(°), 360.0에서 한 바퀴
                        ※ 구현 상 DrawDescription.Rotation.X에서 설정됩니다.
        obj.ry        : Y 축 회전 각도(변경 가능) 단위: 도(°), 360.0에서 한 바퀴
                        ※ 구현 상 DrawDescription.Rotation.Y에서 설정됩니다.
        obj.rz        : Z 축 회전 각도(변경 가능) 단위: 도(°), 360.0에서 한 바퀴
                        ※ 구현 상 DrawDescription.Rotation.Z에서 설정됩니다.
        obj.rxr       : X축 회전 각도(라디안) (변경 가능)
                        ※ rx와 동시에 변경된 경우 이것이 우선 반영됩니다.
        obj.ryr       : Y축 회전 각도(라디안) (변경 가능)
                        ※ ry와 동시에 변경된 경우 이것이 우선 반영됩니다.
        obj.rzr       : Z축 회전 각도(라디안) (변경 가능)
                        ※ rz와 동시에 변경된 경우 이것이 우선 반영됩니다.

        ○ 이미지 크기
        obj.w         : 입력 이미지의 가로 픽셀 수 [읽기 전용]
                        ※ 구현 상 D2D의 GetImageLocalBounds에서 산출된 정수값입니다.
        obj.h         : 입력 이미지의 세로 픽셀 수 [읽기 전용]
                        ※ 구현 상 D2D의 GetImageLocalBounds에서 산출된 정수값입니다.
        obj.hw        : 입력 이미지 가로 픽셀 수의 절반 (= obj.w / 2) [ReadOnly]
        obj.hh        : 입력 이미지 세로 픽셀 수의 절반 (= obj.h / 2) [ReadOnly]
        obj.diagonal  : 입력 이미지의 대각선 길이 (= math.sqrt(w*w + h*h)) [ReadOnly]
        obj.cx        : 이미지의 중심 X 좌표 (= obj.w / 2) [읽기 전용]
                        ※ 픽셀 조작에서 중심 좌표의 참조값입니다.
        obj.cy        : 이미지의 중심 Y 좌표 (= obj.h / 2) [읽기 전용]

        ○ 시간·프레임
        obj.time      : 오브젝트 기준 현재 시간(초) [읽기 전용]
                        ※ 전역 변수 time과 동일한 값입니다.
                        ※ 구현 상 EffectDescription.ItemPosition.Time.TotalSeconds에서 설정됩니다.
        obj.frame     : 오브젝트 기준 현재 프레임 번호 [읽기 전용]
                        ※ 전역 변수 frame과 동일한 값입니다.
                        ※ 구현 상 EffectDescription.ItemPosition.Frame에서 설정됩니다.
        obj.totalframe: 오브젝트의 총 프레임 수 [읽기 전용]
                        ※ 전역 변수 totalframe과 동일한 값입니다.
                        ※ 구현 상 EffectDescription.ItemDuration.Frame에서 설정됩니다.
        obj.framerate : 프레임 레이트(fps) [읽기 전용]
                        ※ 전역 변수 framerate와 동일한 값입니다.
                        ※ 구현 상 EffectDescription.FPS에서 설정됩니다.

        ○ 트랙바
        obj.track0    : 이펙트 패널의 Track0 현재값 [읽기 전용]
                        ※ 애니메이션 보간 후의 값입니다.
        obj.track1    : 이펙트 패널의 Track1 현재값 [읽기 전용]
        obj.track2    : 이펙트 패널의 Track2 현재값 [읽기 전용]
        obj.track3    : 이펙트 패널의 Track3 현재값 [읽기 전용]

        ○ 슬라이더
        obj.slider0   : 이펙트 패널의 Slider0 현재값(정수) [읽기 전용]
        obj.slider1   : 이펙트 패널의 Slider1 현재값(정수) [읽기 전용]
        obj.slider2   : 이펙트 패널의 Slider2 현재값(정수) [읽기 전용]
        obj.slider3   : 이펙트 패널의 Slider3 현재값(정수) [읽기 전용]

        ○ 문자열 파라미터
        obj.text      : 텍스트 입력란의 문자열 [읽기 전용]
        obj.font      : 글꼴 선택란의 글꼴 이름 [읽기 전용]
        obj.dir       : 폴더 선택란의 경로 [읽기 전용]
        obj.file_video ～ obj.file_shader : 각 파일 선택란의 경로 [읽기 전용]

        ○ 레이어
        obj.layer     : 오브젝트가 배치된 레이어 번호 [읽기 전용]
                        ※ 전역 변수 layer와 동일한 값입니다.
                        ※ 구현 상 EffectDescription.Layer에서 설정됩니다.


- obj 테이블 - 변수의 변경과 렌더링 반영 -

        스크립트 실행 후, 다음 변수만이 렌더링 정보로 읽기 복원됩니다(반영됩니다).

        반영되는 변수:
            obj.x, obj.y, obj.z
            obj.ox, obj.oy
            obj.zoom, obj.aspect
            obj.alpha
            obj.rx, obj.ry, obj.rz

        ※ obj.w, obj.h, obj.cx, obj.cy, obj.frame, obj.time,
           obj.totalframe, obj.framerate, obj.track0 ～ obj.track3,
           obj.layer는 읽기 전용이며, 변경해도 렌더링에 반영되지 않습니다.

        출력값 산출 공식:
            ZoomX = obj.zoom * (1.0 + obj.aspect)
            ZoomY = obj.zoom * (1.0 - obj.aspect)
            Opacity = Clamp(obj.alpha / 255.0, 0.0, 1.0)
            Draw = (obj.x, obj.y, obj.z)
            CenterPoint = (obj.ox, obj.oy)
            Rotation = (obj.rx, obj.ry, obj.rz)


- obj 테이블 - 함수 -

        ○ obj.getobject(tag [, frame])
                지정한 태그를 가진 다른 오브젝트의 그리기 정보를 취득합니다.
                ※ 태그는 대상 아이템의 "비고"에 입력한 문자열과 일치합니다.
                ※ 두 번째 인수 frame을 지정하면 해당 타임라인 프레임 시점의 값을
                  취득합니다. 생략하면 현재 프레임의 값을 취득합니다.
                ※ 같은 태그를 가진 아이템이 여러 개인 경우, 대상 프레임에서
                  표시 중인 아이템을 우선하여 반환합니다. 표시 중인 것이 없으면
                  처음으로 일치한 아이템을 반환합니다.
                ※ 취득되는 값은 아이템 자신의 좌표·회전·확대율입니다.
                  그룹 제어·카메라 제어·상위 이펙트에 의한 변형은 포함되지 않습니다.
                ※ 각 값은 obj 테이블과 동일한 단위 체계입니다.
                tag     : 대상 아이템의 비고에 설정한 문자열
                frame   : 취득할 타임라인의 절대 프레임 (생략 가능)
                반환값  : 오브젝트 정보 테이블. 일치하는 아이템이 없으면 nil을 반환합니다.
                          .exist : 대상 프레임에서 표시 중인지 여부 (boolean)
                          .x     : 표시 기준 좌표 X
                          .y     : 표시 기준 좌표 Y
                          .z     : 표시 기준 좌표 Z
                          .zoom  : 확대율 (1.0이 등배)
                          .sx    : X 방향 확대율 (= zoom)
                          .sy    : Y 방향 확대율 (= zoom)
                          .rx    : X축 회전 각도 (도, 항상 0)
                          .ry    : Y축 회전 각도 (도, 항상 0)
                          .rz    : Z축 회전 각도 (도)
                          .rxr   : X축 회전 각도 (라디안, 항상 0)
                          .ryr   : Y축 회전 각도 (라디안, 항상 0)
                          .rzr   : Z축 회전 각도 (라디안)
                          .alpha : 불투명도 (0.0 ～ 255.0, 페이드 반영됨)
                          .layer : 배치 레이어 번호
                예:
                    local funnel = obj.getobject("funnel")
                    if funnel then
                        obj.x = funnel.x
                        obj.y = funnel.y
                    end
                발사 시 좌표로 고정하는 예 (발사원의 비고가 "funnel"인 경우):
                    local fire = timelineframe - frame
                    local src = obj.getobject("funnel", fire)
                    if src then
                        obj.x = src.x
                        obj.y = src.y
                    end

        ○ obj.getpixel(x, y)
                지정 좌표의 픽셀 정보를 취득합니다.
                ※ 최초 호출 시 GPU→CPU로의 이미지 데이터 전송이 발생합니다(처리 부하 수반).
                ※ 픽셀 좌표는 이미지의 좌상단을 (0, 0)으로 하며, 오른쪽·아래 방향이 양수입니다.
                ※ 취득값은 프리멀티플라이드 알파를 역변환한 스트레이트 알파 상당의 값입니다.
                x       : X 좌표 (0 ～ obj.w - 1)
                y       : Y 좌표 (0 ～ obj.h - 1)
                반환값  : r, g, b, a의 4개 값(다중 반환)
                          r = 빨간색 성분 (0.0 ～ 255.0)
                          g = 초록색 성분 (0.0 ～ 255.0)
                          b = 파란색 성분 (0.0 ～ 255.0)
                          a = 알파값 (0.0 ～ 255.0)
                          ※ 범위 밖의 좌표를 지정하면 (0, 0, 0, 0)을 반환합니다.
                예: r, g, b, a = obj.getpixel(10, 20)

        ○ obj.setpixel(x, y, r, g, b [, a])
                지정 좌표의 픽셀 정보를 덮어씁니다.
                ※ 호출 시 자동으로 픽셀 데이터가「변경됨」으로 표시되며,
                  스크립트 종료 후 변경된 이미지가 출력됩니다.
                ※ 내부에서는 프리멀티플라이드 알파 형식(BGRA)으로 보유합니다.
                   setpixel 시에 premultiplied 변환이 이루어집니다.
                x       : X 좌표 (0 ～ obj.w - 1)
                y       : Y 좌표 (0 ～ obj.h - 1)
                r       : 빨간색 성분 (0.0 ～ 255.0)
                g       : 초록색 성분 (0.0 ～ 255.0)
                b       : 파란색 성분 (0.0 ～ 255.0)
                a       : 알파값 (생략 시 255.0)
                예: obj.setpixel(10, 20, 255, 0, 0, 128)

        ○ obj.getpixeldata()
                전체 픽셀 데이터에 일괄 접근하기 위한 프록시 오브젝트를 반환합니다.
                ※ getpixel/setpixel에 비해 빠르게 픽셀을 조작할 수 있습니다.
                ※ 최초 호출 시 GPU→CPU로의 이미지 데이터 전송이 발생합니다(처리 부하 수반).
                반환값  : PixelDataProxy 오브젝트
                          .width  : 이미지의 가로 픽셀 수 (= obj.w)
                          .height : 이미지의 세로 픽셀 수 (= obj.h)
                          :get(index)        : 인덱스 지정으로 채널값 취득
                          :set(index, value) : 인덱스 지정으로 채널값 설정

                인덱스 사양:
                    인덱스는 1부터 시작합니다(Lua 관례에 따름).
                    채널 순서는 R, G, B, A의 반복입니다.
                    pixel(x, y)의 R 채널 인덱스 =
                        (y * obj.w + x) * 4 + 1
                    pixel(x, y)의 G 채널 인덱스 =
                        (y * obj.w + x) * 4 + 2
                    pixel(x, y)의 B 채널 인덱스 =
                        (y * obj.w + x) * 4 + 3
                    pixel(x, y)의 A 채널 인덱스 =
                        (y * obj.w + x) * 4 + 4

                    ※ 알파(채널 4)를 변경하면 RGB도 비율 변환됩니다.
                    ※ RGB 채널을 변경하면 premultiplied 변환이 이루어집니다.
                    ※ 취득값은 get(index)와 마찬가지로 스트레이트 알파 상당입니다.

                예:
                    local pd = obj.getpixeldata()
                    local idx = (10 * obj.w + 20) * 4 + 1  -- (x=20, y=10)의 R 채널
                    local r = pd:get(idx)
                    pd:set(idx, 128)

        ○ obj.putpixeldata()
                호환성을 위해 정의된 빈 구현 함수입니다(no-op).
                ※ AviUtl 스크립트에서는 getpixeldata/putpixeldata 조합으로 사용하지만,
                  이 플러그인에서는 setpixel/set에 의한 변경이 자동으로 반영되므로,
                  이 함수를 호출할 필요가 없습니다.

        ○ obj.rand(st, ed [, seed [, frame]])
                지정 범위 st ～ ed의 정수를 반환하는 결정론적 난수입니다.
                ※ seed와 frame이 같으면 항상 같은 값을 반환합니다.
                ※ seed 생략 시 0으로, frame 생략 시 현재 프레임(obj.frame)을 사용합니다.
                ※ 기존 엔진(MoonSharp)과 고속 런타임(LuaJIT)에서 같은 값을 반환합니다.
                  엔진에 의존하지 않는 소수 난수가 필요하면 anim.rand를 사용하세요.
                st      : 범위 하한 (소수는 버림하여 정수화)
                ed      : 범위 상한 (소수는 버림하여 정수화)
                seed    : 난수 시드 (생략 가능, 기본값 0)
                frame   : 변화에 사용하는 프레임 (생략 가능, 기본값 obj.frame)
                반환값  : st ～ ed의 정수
                예: obj.ox = obj.ox + obj.rand(-10, 10, 1, obj.frame)

        ○ obj.load(대상, ...)
                오브젝트 버퍼를 생성한 내용으로 교체합니다. "figure"(도형) /
                "text"(텍스트) / "image"(이미지 파일) / "movie"(동영상 파일)를
                지원합니다. 로드 후 obj.w / obj.h 등은 생성 크기로 갱신되며, 로드
                직후의 getpixel로도 참조할 수 있습니다. 기존 엔진과 고속 엔진
                양쪽에서 동작합니다.

                obj.load("figure", 이름, 색, 크기 [, 라인 [, 종횡비]])
                        도형을 생성하여 버퍼를 교체합니다(중심 배치).
                        이름   : 円 / 四角形 / 三角形 / 五角形 / 六角形 / 星形
                                 (영어 circle, rect, triangle, pentagon, hexagon,
                                 star 도 사용 가능).
                        색     : 채움 색 0xRRGGBB.
                        크기   : 기준 크기(픽셀).
                        라인   : 외곽선 두께(생략 가능). 0이면 채움.
                        종횡비 : 왜곡 -1.0 ～ 1.0(생략 가능).
                        예: obj.load("figure", "円", 0xff3366, 200)

                obj.load("text", 문자열)
                        obj.setfont로 지정한 글꼴・크기・색으로 문자열을 그려
                        버퍼를 교체합니다(DirectWrite로 그림).
                        예: obj.setfont("Yu Gothic UI", 48); obj.load("text", "자막")

                obj.load("image", 경로)
                        이미지 파일을 읽어 버퍼를 교체합니다.
                        ※ 안전을 위해 선두 헤더를 검증하여 진짜 이미지
                          (PNG / JPEG / GIF / BMP / TIFF / WebP)만 읽습니다.
                        예: obj.load("image", "C:\\img\\logo.png")

                obj.load("movie", 경로 [, 시간])
                        동영상 파일의 지정 시간 프레임을 읽어 버퍼를 교체합니다
                        (Media Foundation으로 디코딩).
                        시간 : 초. 생략 시 obj.time.
                        ※ 안전을 위해 선두 헤더를 검증하여 진짜 동영상
                          (MP4 / MOV / MKV / WebM / AVI / FLV / ASF)만 읽습니다.
                        예: obj.load("movie", "C:\\mov\\clip.mp4", obj.time)

        ○ obj.setfont(글꼴 [, 크기 [, 스타일 [, 색]]])
                obj.load("text")에서 사용할 글꼴을 설정합니다.
                스타일 : 비트 플래그. 1=굵게, 2=기울임, 3=굵게+기울임.
                색     : 0xRRGGBB.
                예: obj.setfont("Meiryo", 64, 1, 0xffffff)

        ○ obj.draw([ox, oy, oz [, 확대율 [, 불투명도 [, 종횡비]]]])
                현재 오브젝트 이미지를 누적 버퍼에 합성하여 그립니다.
                여러 번 호출하면 겹쳐서 그려집니다(잔상・파티클 등).
                ox, oy, oz : 중심으로부터의 오프셋(기본 0).
                확대율     : 1.0이 등배(기본 1.0).
                불투명도   : 0.0 ～ 1.0(기본 1.0).
                종횡비     : -1.0 ～ 1.0(기본 0).
                ※ 호출하면 표준 그리기가 억제되고 합성 결과가 출력됩니다.

        ○ obj.drawpoly(x0,y0,z0, x1,y1,z1, x2,y2,z2, x3,y3,z3
                        [, u0,v0, u1,v1, u2,v2, u3,v3 [, 불투명도]])
                4개 정점으로 지정한 사각형에 오브젝트 이미지를 텍스처로 그립니다.
                정점 순서는 좌상→우상→우하→좌하. UV 생략 시 전체 이미지를 사용합니다.
                ※ UV를 생략하고 13번째 인수만 준 경우, 그 값을 불투명도로 사용합니다
                  (좌표 12개 + 불투명도).
                ※ 아핀 변환으로 매핑합니다(평행사변형은 정확, 비평행사변형은 근사).

        ○ obj.copybuffer(대상, 원본)
                버퍼 간에 이미지를 복사합니다. 식별자의 첫 글자로 판정합니다.
                  "obj"      : 오브젝트 이미지 버퍼
                  "tmp"      : 임시 버퍼
                  "cache:이름" : 명명 캐시(프레임 간 유지)
                예: obj.copybuffer("tmp", "obj")  -- 저장
                    obj.copybuffer("obj", "tmp")  -- 복원
                ※ "frm"(프레임 버퍼 / 배경)은 YMM4의 구조상 가져올 수 없습니다.

        ○ obj.getvalue(대상)
                obj의 현재 값을 가져옵니다(obj.x, obj.track0 등과 동일).
                대상   : "x","y","z","ox","oy","rx","ry","rz","zoom","aspect",
                         "alpha","sx","sy","track0"～"track3","time","frame" 등.
                반환값 : 숫자(알 수 없는 대상은 0).

        ○ obj.setoption(이름, 값) / obj.getoption(이름)
                옵션을 설정 / 가져옵니다. 지원하지 않는 항목도 받아들여
                스크립트를 중단시키지 않습니다(getoption은 설정값을 반환, 미설정은 0).
                현재 그리기에 반영되는 항목:
                  "antialias" : 0이면 최근접, 그 외는 선형 보간(obj.draw / obj.drawpoly).
                  "blend" : 합성 모드 번호(obj.draw / obj.drawpoly). YMM4 속성
                    패널의 "합성 모드" 드롭다운과 동일한 순서(0부터 시작):
                    0=보통,1=디졸브,2=비교(어두움),3=곱하기,4=색상번,
                    5=구워내기(선형),6=비교(밝음）,7=화면,8=색상 닷지,
                    9=선형 닷지 ‐ 가산,10=가산,11=오버레이,12=소프트 라이트,
                    13=하드 라이트,14=선명한 라이트,15=리니어 라이트,16=핀 라이트,
                    17=하드 믹스,18=차분,19=제외,20=감산,21=디비전,22=색조,
                    23=채도,24=컬러,25=밝기,26=색상 비교(밝음),27=배경,
                    28=색상 비교(어두운),29=삭제,30=배경에서 클리핑,
                    31=겹치지 않는 부분만,32=색상 반전 마스크.
                    범위를 벗어나거나 잘못된 값은 0(보통)으로 처리합니다.
                  "draw_state" : 이 이펙트가 작업 이미지를 출력할지를 불리언으로
                    정합니다. true는 작업 이미지를 출력하고, false는 입력을 그대로
                    통과시킵니다. obj.draw나 setpixel을 쓰지 않고 작업 이미지를
                    출력하거나, 반대로 출력을 막을 때 사용합니다. 두 엔진 모두에서
                    동작합니다.
                  "drawtarget" : obj.draw / obj.drawpoly의 그리기 대상을 바꿉니다.
                    "framebuffer"(기본)는 프레임 버퍼에 합성합니다. "tempbuffer"는
                    작업용 tmp 버퍼에 즉시 합성합니다.
                    obj.setoption("drawtarget", "tempbuffer", w, h)의 w, h는 tmp
                    버퍼를 초기화하는 크기이며, 생략하면 초기화하지 않습니다. tmp에
                    그린 결과는 obj.copybuffer("obj", "tmp")로 작업 이미지에 되돌릴
                    수 있습니다. tempbuffer로의 합성은 보통(소스 오버)이며, blend는
                    framebuffer 그리기에 적용됩니다. 두 엔진 모두에서 동작합니다.

        ○ obj.pixeloption(이름, 값)
                픽셀 조작 옵션을 상태로 보존합니다.

        ○ obj.setanchor(이름, 개수 [, 옵션, ..])
                프리뷰에 앵커 포인트를 표시하고 마우스 드래그로 좌표를 조정할 수
                있게 합니다. 조정한 좌표는 프로젝트에 저장됩니다.
                이름 : 좌표를 받을 Lua 변수 이름을 문자열로 지정합니다. 변수는
                    pos={x0,y0,x1,y1,…} 배열이며, "xyz" 지정 시
                    {x0,y0,z0,x1,y1,z1,…}가 됩니다. 좌표는 오브젝트 중심이
                    원점입니다.
                개수 : 앵커 포인트 수를 지정합니다. 최대 32개입니다.
                옵션 : 문자열로 열거할 수 있습니다.
                    "line" : 앵커 포인트를 선으로 잇습니다.
                    "loop" : 앵커 포인트를 선으로 잇고 한 바퀴 닫습니다.
                    "star" : 각 앵커 포인트를 오브젝트 중심과 선으로 잇습니다.
                    "arm"  : 오브젝트 중심에서 각 앵커 포인트로 차례로 잇습니다.
                    "xyz"  : 앵커 포인트를 3D 좌표로 다룹니다. 기본은 2D입니다.
                설치한 앵커 포인트 수를 반환합니다. 두 엔진 모두에서 동작합니다.
                이름에 "track"을 지정하는 트랙바 연동은 현재 미지원이며 0을 반환합니다.
                예:
                  num = obj.track0
                  obj.setanchor("pos", num, "loop")
                  for i = 0, num - 1 do
                    obj.draw(pos[i * 2 + 1], pos[i * 2 + 2])
                  end

- anim 테이블 -

        애니메이션 계산에 편리한 순수 유틸리티 함수/상수 그룹입니다.

        anim.tau                : 원주율의 2배 (math.pi * 2)
        anim.e                  : 오일러 수 (math.exp(1))
        anim.phi                : 황금비 ((1 + math.sqrt(5)) / 2)
        anim.sqrt2              : 2의 제곱근 (math.sqrt(2))
        anim.lerp(a, b, t)      : a와 b를 t(0.0~1.0)로 선형 보간
        anim.smoothstep(e0, e1, x): e0과 e1 사이를 매끄럽게 보간
        anim.smootherstep(e0, e1, x): e0과 e1 사이의 더 부드러운 에르미트 보간 (2차 도함수까지 연속)
        anim.clamp(v, lo, hi)   : 값 v를 lo~hi 범위로 제한
        anim.map(v, a1, b1, a2, b2): a1~b1 범위의 값 v를 a2~b2 범위로 매핑
        anim.norm(v, lo, hi)    : 값 v가 lo~hi 범위에서 차지하는 비율(0.0~1.0) 반환
        anim.wrap(v, lo, hi)    : 값 v를 lo~hi 범위 내에서 루프시킴
        anim.pingpong(t, len)   : 0~len 범위를 왕복하는 값
        anim.sign(v)            : 값 v의 부호 반환 (양수=1, 음수=-1, 0=0)
        anim.oscillate(t, lo, hi, freq): freq 주파수로 lo~hi를 매끄럽게 왕복(사인파)
        anim.triangle(t, freq)  : freq 주파수로 0.0~1.0을 왕복하는 삼각파
        anim.square(t, freq)    : freq 주파수로 1.0과 0.0을 전환하는 구형파
        anim.duration(t, dur)   : 시간 dur에 대한 현재 비율(0.0~1.0, clamp 적용)
        anim.delay(t, d)        : 시간 d 경과 후 시작되는 시간 (0 이하는 0)
        anim.ease_in(t)         : 2차 이즈 인(가속)
        anim.ease_out(t)        : 2차 이즈 아웃(감속)
        anim.ease_in_out(t)     : 2차 이즈 인/아웃 (가속 후 감속)
        anim.elastic(t)         : 튕기는 듯한 스프링 이징
        anim.back(t)            : 범위를 살짝 넘었다가 돌아오는 이징
        anim.step(edge, x)      : x가 edge 이상이면 1.0, 미만이면 0.0 반환
        anim.fract(v)           : 값 v의 소수 부분만 반환
        anim.bounce(t)          : 바운스 이징
        anim.hsv_to_rgb(h, s, v): HSV (색상 0~360, 채도 0~1, 명도 0~1)를 RGB로 변환
        anim.rgb_to_hsv(r, g, b): RGB (0~255)를 HSV로 변환
        anim.noise(x, y, z)     : 1차원~3차원의 부드러운 노이즈 (0.0 ~ 1.0)
        anim.rand(min, max, s)  : 시드값 s에 기반한 안정적인 난수 생성
        anim.len(x, y)          : 벡터 (x, y)의 길이(노름)를 가져옵니다
        anim.dist(x1, y1, x2, y2): 두 점 (x1, y1)과 (x2, y2) 사이의 거리를 가져옵니다
        anim.dot(x1, y1, x2, y2): 두 벡터 (x1, y1)과 (x2, y2)의 내적을 가져옵니다
        anim.normalize(x, y)    : 벡터 (x, y)를 정규화하여 방향 벡터(Tuple)를 반환합니다
        anim.polar(r, a)        : 반지름 r, 각도 a(도)에서 (x, y) 이동량(Tuple)을 가져옴
        anim.rotate(x, y, a)    : 좌표 (x, y)를 a(도)만큼 회전시킨 좌표(Tuple)를 가져옴
        anim.bezier(t, p0, p1, p2, p3): t(0.0~1.0)에서의 3차 베지어 곡선 값을 계산

- AviUtl 스크립트 부분 호환 -

        AviUtl 애니메이션 효과 스크립트(*.anm 계열) 중 좌표·확대축소·회전·
        불투명도의 변형과 난수를 사용하는 범위를, 추가 수정 없이 실행할 수 있는
        호환 계층입니다. 스크립트에 AviUtl 고유 표기가 포함되지 않은 경우 이
        호환 계층은 전혀 개입하지 않습니다(기존 스크립트의 동작은 변하지 않습니다).

        ○ 스크립트 헤더 디렉티브 해석
                주석 형식의 파라미터 선언을 해석하여, 스크립트 본문이 참조하는
                변수를 실행 전에 정의합니다.
                --dialog:항목명,local 변수=기본값;…
                        각 항목의 선언부(local 변수=기본값 등)를 본문 앞에 펼칩니다.
                        AviUtl에서는 대화 상자에서 편집하는 값이지만, 여기서는
                        기본값으로 동작합니다.
                --color:0xRRGGBB
                        변수 color를 지정한 색으로 정의합니다.
                --check0:항목명,기본값
                        obj.check0 (0～3)을 기본값으로 정의합니다.
                --param:임의의 Lua 문
                        작성한 Lua 문을 그대로 본문 앞에 펼칩니다.
                --track0: ～ --track3:
                        트랙바 값은 기존과 같이 obj.track0 ～ obj.track3에서
                        참조합니다. 헤더의 라벨·범위·기본값은 UI에 반영되지 않습니다.

        ○ @를 통한 복수 섹션
                하나의 스크립트에 @이름으로 여러 효과가 정의된 경우, 첫 번째
                섹션만 실행합니다.

        ○ obj.rand(st, ed [, seed [, frame]])
                AviUtl 호환의 결정론적 정수 난수입니다. 자세한 내용은
                "obj 테이블 - 함수"를 참조하세요.

        ○ 그리기·합성·로드
                obj.draw / obj.drawpoly에 의한 그리기 합성, obj.copybuffer에 의한
                버퍼 조작, obj.load("figure"/"text"/"image"/"movie")에 의한 생성·로드,
                obj.setfont, obj.effect, obj.getvalue, 그리고 obj.setoption /
                obj.getoption / obj.pixeloption을 지원합니다. 자세한 내용은
                "obj 테이블 - 함수"를 참조하세요.

        ○ obj.effect의 AviUtl / AviUtl2 호환 맵
                obj.effect("이펙트 이름", "파라미터 이름", 값, ...)은 먼저 호환 맵을
                참조합니다. AviUtl / AviUtl2의 이름으로 선언된 이펙트는 값이 선언된
                배율·오프셋 변환을 거쳐 YMM4의 속성에 반영됩니다. 맵에 선언되지 않은
                이름은 YMM4의 이펙트 이름이나 키워드로 그대로 해석되므로 기존 호출도
                그대로 동작합니다.
                이 맵은 YMM4 자체의 AviUtl 내보내기(.exo 출력)의 역으로 만들어졌기 때문에
                왕복해도 일치합니다. 대상 엔진은 --!aviutl(기본) / --!aviutl2로
                전환합니다.
                맵에 선언된 이펙트 이름과 주요 파라미터는 다음과 같습니다.
                  ぼかし: 範囲, サイズ固定
                  境界ぼかし: 範囲
                  方向ブラー: 範囲, 角度, サイズ固定
                  放射ブラー: 範囲, X, Y, サイズを固定
                  閃光: 範囲, X, Y, サイズを固定
                  クロマキー: 色相範囲, 境界補正
                  クリッピング: 上, 下, 左, 右, 中心の位置を変更
                  斜めクリッピング: 中心X, 中心Y, 角度, ぼかし, 幅
                  領域拡張: 上, 下, 左, 右
                  座標: X, Y, Z
                  回転: X, Y, Z
                  リサイズ: 拡大率, X, Y, 補間なし
                  画像ループ: 横回数, 縦回数
                  極座標変換: 中心幅, 渦巻
                  グラデーション: 強さ, 中心X, 中心Y, 角度, 幅
                  単色化: 強さ, 輝度を保持する
                  縁取り: サイズ, ぼかし
                  反転: -
                  モザイク: サイズ
                  ディスプレイスメントマップ: ぼかし
                  動画ファイル合成: X, Y, 拡大率
                  色調補正: 明るさ, ｺﾝﾄﾗｽﾄ, 色相, 輝度, 彩度
                  色ずれ: ずれ幅, 角度, 強さ
                色調補正 와 色ずれ 는 커뮤니티 플러그인의 이펙트에 대응합니다.

        다음 기능은 이 호환 계층의 범위 밖입니다.
        이를 호출하는 스크립트는 의도대로 동작하지 않습니다.
                × require에 의한 모듈 로드 및 io / dofile 등의 임의 파일 접근
                  (obj.load의 이미지 / 동영상 로드는 헤더 검증과 함께 지원).
                × obj.setanchor, scene.set / scene.get 등.


- 미지원·제한 사항 -

        × AviUtl2의 obj.rand1(), obj.mes(), obj.setanchor() 등은 지원하지 않습니다.
        × AviUtl2의 scene 테이블의 scene.set(), scene.get() 등의 함수는 지원하지 않습니다.

        × 스크립트에서의 임의 파일 접근(require, dofile, io 등)은 지원하지 않습니다.
          obj.load("image"/"movie")만 선두 헤더 검증으로 진짜 미디어로 확인된 파일을
          읽습니다.
        × obj.z를 변경하면 렌더링에 반영되지만(ReadBack에 포함),

        × 스크립트 타임아웃(5초)을 초과하면 실행이 강제 종료됩니다.
          무한 루프가 발생하지 않도록 구현 시 주의가 필요합니다.
        × AviUtl2의 헤더 디렉티브는 부분적으로만 지원합니다. --dialog / --color /
          --check / --param의 기본값은 해석되지만("AviUtl 스크립트 부분 호환" 참조),
          --track@의 라벨·범위·기본값은 UI에 반영되지 않으며 값은 Track0 ～ Track3을 사용합니다.
