- 關於 Lua 腳本特效 -

        YukkuriMovieMaker4 的「Lua 腳本」視訊特效使用
        Lua 語言編寫腳本。
        腳本由隨附的高速執行環境（LuaJIT）執行，
        在其不可用的環境下改由 MoonSharp（相容 Lua 5.2）執行。

        * 本特效為獨立實作，與 AviUtl2 的腳本規範不同。
        * 與 AviUtl2 的腳本檔案（*.anm2 等）為不同規範。
        * 由於 IsAviUtlSupported = false，不支援面向 AviUtl 的 EXO 輸出。


- 注意事項 -

        ○ 腳本執行引擎為 LuaJIT（預設）與 MoonSharp（相容 Lua 5.2）。
        ○ 可用的標準函式庫僅限以下內容：
            table, string, math, bit32
            * io、os、debug、ffi、require 等不可使用。
        ○ 腳本逾時時間為 5000ms（5 秒）。
            逾時時，執行將被強制終止並視為錯誤。
        ○ 腳本無法存取檔案系統（包括 require）。
        ○ 不支援從腳本內部呼叫外部腳本。
        ○ 像素操作（getpixel/setpixel/getpixeldata）會存取實際輸入影像，
            因此僅在首次存取時發生 GPU→CPU 的資料傳輸。
            不需要時省略呼叫可加快處理速度。


- 執行引擎與自動加速 -

        腳本會被解析語法，並自動分配到最合適的引擎。
        未指定指令的腳本由隨附的高速執行環境（LuaJIT）執行，
        自像素完結的像素映射會進一步轉換為更快的 CPU 核心。
        兩者的結果均與 MoonSharp 一致。

        ○ 自動分配（未指定指令時）
                · 自像素完結的像素映射（見下方條件）→ CPU 核心（高速、精確）
                · 上述以外 → 原生（LuaJIT，僅在已部署時）
                · 未部署 native 或腳本為空 → MoonSharp

        ○ 引擎指定指令（腳本首行的註解）
                --!cpu       ：CPU 核心。執行時編譯為運算式樹，
                               在所有 CPU 核心上並行執行（與 MoonSharp 完全一致）。
                --!gpu       ：GPU 核心。作為 Direct2D 像素著色器執行，
                               排除 GPU→CPU 傳輸，速度最快。
                               因 float32 精度，結果可能有極小差異（需主動選用）。
                --!native    ：LuaJIT（獨立程序）。高速且與 MoonSharp 一致。
                               需要部署 native 資料夾。
                --!moonsharp ：強制使用傳統的 MoonSharp。
                ※ 指令不可用/不支援時，會自動回退到較低的引擎
                  （輸出不會損壞）。

        ○ 可成為高速核心的語法（純像素映射）
                只有以下形式的雙重迴圈才會被核心化。
                    for y = 0, obj.h - 1 do
                        for x = 0, obj.w - 1 do
                            local r, g, b, a = obj.getpixel(x, y)
                            obj.setpixel(x, y, r2, g2, b2 [, a2])
                        end
                    end
                · 主體中僅可使用四則運算、math.*、比較、閾值寫法
                  「條件 and 值1 or 值2」以及 local 變數。
                · 僅可參照正在處理的自像素（getpixel(x, y)）
                  以及 obj/scene/time 等值。
                · 含有鄰域參照（模糊等）、寫入其他座標、getpixeldata、
                  getobject、表/字串操作、if 陳述式等的腳本不會被核心化，
                  會自動由傳統引擎執行。

        範例（讓白色圖形變紅）：
                --!gpu
                for y = 0, obj.h - 1 do
                    for x = 0, obj.w - 1 do
                        local r, g, b, a = obj.getpixel(x, y)
                        obj.setpixel(x, y, r, g / 2, b / 2, a)
                    end
                end


- 參數 -

        特效面板包含以下 4 個軌道條（支援動畫）。
        各值在腳本中可透過 obj.track0 ～ obj.track3 參照。

        Track0：參數 0（預設值：0，範圍：最小值 ～ 最大值）
        Track1：參數 1（預設值：0，範圍：最小值 ～ 最大值）
        Track2：參數 2（預設值：0，範圍：最小值 ～ 最大值）
        Track3：參數 3（預設值：0，範圍：最小值 ～ 最大值）

        * 每個軌道條均支援動畫滑桿（含插值）。
        * 可透過腳本編輯器工具列匯入/匯出 .lua 檔案。
        * 核取方塊（obj.check0～obj.check3）與顏色（color）會在腳本本體參照它們時
          自動顯示於特效面板中（軌道條始終顯示）。此通用用法不帶範圍・預設值宣告，
          因此請在腳本內處理取值範圍。AviUtl 的 --track / --check / --color 標頭宣告顯示仍可照常使用。

        共有 4 個數值滑桿，各值可透過 obj.slider0 ～ obj.slider3 參照。
        取值為整數，不處理小數。

        Slider0 : 整數滑桿 (範圍: 1 ～ 10)
        Slider1 : 整數滑桿 (範圍: 0 ～ 100)
        Slider2 : 整數滑桿 (範圍: 0 ～ 1000)
        Slider3 : 整數滑桿 (範圍: -1000 ～ 1000)

        亦可使用字串參數，各值透過以下名稱參照。

        obj.text          : 文字輸入框的字串
        obj.font          : 字型選擇框的字型名
        obj.dir           : 資料夾選擇框的路徑
        obj.file_video    : 影片檔案的路徑
        obj.file_audio    : 音訊檔案的路徑
        obj.file_image    : 圖片檔案的路徑
        obj.file_project  : 專案檔案的路徑
        obj.file_mp4      : MP4 檔案的路徑
        obj.file_exo      : EXO 檔案的路徑
        obj.file_subtitle : 字幕檔案的路徑
        obj.file_shader   : 著色器檔案的路徑

        ※滑桿與字串參數在腳本本體參照它們時會自動顯示於特效面板中。
        ※檔案選擇的路徑可傳給 obj.load 用於載入圖片・影片。
        ※這些參數在 MoonSharp 與原生兩種引擎中皆可參照。


- 全域變數 -

        以下變數可在腳本的全域範圍內直接參照。

        time          : 相對於物件的當前時間（秒）[唯讀]
                        * 與 obj.time 的值相同。
        frame         : 相對於物件的當前影格編號 [唯讀]
                        * 與 obj.frame 的值相同。
        totalframe    : 物件的總影格數 [唯讀]
                        * 與 obj.totalframe 的值相同。
        framerate     : 影格率（fps）[唯讀]
                        * 與 obj.framerate 的值相同。
        timelineframe : 相對於整個時間軸的當前影格編號 [唯讀]
                        * obj.frame 是從物件起點的相對影格，
                          而此值是從時間軸起點的絕對影格。
        timelinetime  : 相對於整個時間軸的當前時間（秒）[唯讀]
        layer         : 物件所在的圖層編號 [唯讀]
                        * 與 obj.layer 的值相同。
        obj           : 物件資訊表（參見下文「obj 表」）
        scene         : 場景資訊表（參見下文「scene 表」）
        ymm4          : YMM4專有的元數據表 (參見下文「ymm4 表」)
        anim          : 動畫專用工具表 (參見下文「anim 表」)


- scene 表 -

        scene.width   : 場景（專案）的水平解析度（像素）[唯讀]
        scene.height  : 場景（專案）的垂直解析度（像素）[唯讀]

        scene.set(name, value)
                將值儲存到以場景為單位的共享區域。
                同一場景內的所有 Lua 腳本效果都可以跨項目、跨影格透過
                scene.get(name) 讀取該值。
                name  : 值的名稱（字串）
                value : 要儲存的值（數值、字串、布林值）。傳入 nil 時刪除。
                * 數值、字串、布林值以外的型別（表、函式等）不會被儲存。
                * 名稱和字串值按 UTF-8 計算超過 4095 位元組的部分將被截斷。
                例：scene.set("shared_x", obj.x)
        scene.get(name)
                取得透過 scene.set() 儲存的值。未設定的名稱回傳 nil。
                例：local x = scene.get("shared_x") or 0
        * 值按場景相互獨立，在 YMM4 結束前一直保留在記憶體中
          （不會儲存到專案檔）。
        * 項目之間的求值順序沒有保證，請避免依賴同一影格內寫入與讀取順序的
          設計。


- obj 表 - 變數 -

        以下變數保存物件當前的繪製資訊。
        * 未標注「唯讀」的變數可在腳本中修改，修改後的值將反映至繪製中。

        ○ 座標·偏移
        obj.x         : 顯示基準座標 X（可修改）
                        * 對應 AviUtl2 規範的「顯示基準座標 X」。
                        * 內部由 DrawDescription.Draw.X 設定，
                          腳本執行後更改內容才會生效。
        obj.y         : 顯示基準座標 Y（可修改）
                        * 內部由 DrawDescription.Draw.Y 設定。
        obj.z         : 顯示基準座標 Z（可修改）
                        * 內部由 DrawDescription.Draw.Z 設定。
        obj.ox        : 中心偏移 X（可修改）
                        * 內部由 DrawDescription.CenterPoint.X (inDesc) 設定。
                        * 對應 AviUtl2 規範的「從基準座標的相對座標 X (obj.ox)」，
                          但在本外掛中作為 CenterPoint（中心點偏移）處理。
        obj.oy        : 中心偏移 Y（可修改）
                        * 內部由 DrawDescription.CenterPoint.Y (inDesc) 設定。

        ○ 縮放·變形
        obj.zoom      : 縮放比例（可修改）
                        * 等倍為 1.0 而非 0.0（相容 AviUtl2）。
                        * 內部由 ZoomX 與 ZoomY 的平均值（zoomAvg）計算得出。
                        * 輸出時以 obj.zoom * (1 ± obj.aspect) 展開為 X/Y 各自的 Zoom。
        obj.aspect    : 長寬比（可修改），範圍：-1.0 ～ 1.0
                        * 正值在 X 方向縮小，負值在 Y 方向縮小。
                        * 內部由 (ZoomX - ZoomY) / (ZoomX + ZoomY) 計算得出。
        obj.sx        : X方向縮放率 (可修改) 1.0為100%
                        ※ YMM4專有的精確獨立縮放控制。
                        ※ 優先於 zoom/aspect 的修改生效。
        obj.sy        : Y方向縮放率 (可修改) 1.0為100%
                        ※ YMM4專有的精確獨立縮放控制。
                        ※ 優先於 zoom/aspect 的修改生效。
        obj.alpha     : 不透明度（可修改），範圍：0.0 ～ 255.0
                        * AviUtl2 為 0.0～1.0，本外掛為 0.0～255.0。
                        * 內部由 DrawDescription.Opacity * 255 設定。
                        * 輸出時除以 255 並正規化為 0.0～1.0（含 Clamp）。
                        * 預設腳本設定了 obj.alpha = math.min(time * 255, 255)，
                          作為淡入的預設實作。

        ○ 旋轉
        obj.rx        : X 軸旋轉角度（可修改），單位：度（°），360.0 為一圈
                        * 內部由 DrawDescription.Rotation.X 設定。
        obj.ry        : Y 軸旋轉角度（可修改），單位：度（°），360.0 為一圈
                        * 內部由 DrawDescription.Rotation.Y 設定。
        obj.rz        : Z 軸旋轉角度（可修改），單位：度（°），360.0 為一圈
                        * 內部由 DrawDescription.Rotation.Z 設定。
        obj.rxr       : X軸旋轉角度(弧度) (可修改)
                        ※ 若與rx同時修改，將優先反映此值。
        obj.ryr       : Y軸旋轉角度(弧度) (可修改)
                        ※ 若與ry同時修改，將優先反映此值。
        obj.rzr       : Z軸旋轉角度(弧度) (可修改)
                        ※ 若與rz同時修改，將優先反映此值。

        ○ 影像尺寸
        obj.w         : 輸入影像的水平像素數 [唯讀]
                        * 內部為從 D2D 的 GetImageLocalBounds 計算得出的整數值。
        obj.h         : 輸入影像的垂直像素數 [唯讀]
                        * 內部為從 D2D 的 GetImageLocalBounds 計算得出的整數值。
        obj.hw        : 輸入圖像水平像素數的一半 (= obj.w / 2) [ReadOnly]
        obj.hh        : 輸入圖像垂直像素數的一半 (= obj.h / 2) [ReadOnly]
        obj.diagonal  : 輸入影像的對角線長度 (= math.sqrt(w*w + h*h)) [ReadOnly]
        obj.cx        : 影像中心 X 座標（= obj.w / 2）[唯讀]
                        * 像素操作中中心座標的參考值。
        obj.cy        : 影像中心 Y 座標（= obj.h / 2）[唯讀]

        ○ 時間·影格
        obj.time      : 相對於物件的當前時間（秒）[唯讀]
                        * 與全域變數 time 的值相同。
                        * 內部由 EffectDescription.ItemPosition.Time.TotalSeconds 設定。
        obj.frame     : 相對於物件的當前影格編號 [唯讀]
                        * 與全域變數 frame 的值相同。
                        * 內部由 EffectDescription.ItemPosition.Frame 設定。
        obj.totalframe: 物件的總影格數 [唯讀]
                        * 與全域變數 totalframe 的值相同。
                        * 內部由 EffectDescription.ItemDuration.Frame 設定。
        obj.framerate : 影格率（fps）[唯讀]
                        * 與全域變數 framerate 的值相同。
                        * 內部由 EffectDescription.FPS 設定。

        ○ 軌道條
        obj.track0    : 特效面板 Track0 的當前值 [唯讀]
                        * 動畫插值後的值。
        obj.track1    : 特效面板 Track1 的當前值 [唯讀]
        obj.track2    : 特效面板 Track2 的當前值 [唯讀]
        obj.track3    : 特效面板 Track3 的當前值 [唯讀]

        ○ 滑桿
        obj.slider0   : 特效面板 Slider0 的當前值(整數) [唯讀]
        obj.slider1   : 特效面板 Slider1 的當前值(整數) [唯讀]
        obj.slider2   : 特效面板 Slider2 的當前值(整數) [唯讀]
        obj.slider3   : 特效面板 Slider3 的當前值(整數) [唯讀]

        ○ 字串參數
        obj.text      : 文字輸入框的字串 [唯讀]
        obj.font      : 字型選擇框的字型名 [唯讀]
        obj.dir       : 資料夾選擇框的路徑 [唯讀]
        obj.file_video ～ obj.file_shader : 各檔案選擇框的路徑 [唯讀]

        ○ 圖層
        obj.layer     : 物件所在的圖層編號 [唯讀]
                        * 與全域變數 layer 的值相同。
                        * 內部由 EffectDescription.Layer 設定。


- obj 表 - 變數修改與繪製反映 -

        腳本執行後，僅以下變數會被回讀至繪製資訊中（生效）。

        生效的變數：
            obj.x, obj.y, obj.z
            obj.ox, obj.oy
            obj.zoom, obj.aspect
            obj.alpha
            obj.rx, obj.ry, obj.rz

        * obj.w、obj.h、obj.cx、obj.cy、obj.frame、obj.time、
          obj.totalframe、obj.framerate、obj.track0 ～ obj.track3、
          obj.layer 為唯讀，修改後不會反映至繪製中。

        輸出值計算公式：
            ZoomX = obj.zoom * (1.0 + obj.aspect)
            ZoomY = obj.zoom * (1.0 - obj.aspect)
            Opacity = Clamp(obj.alpha / 255.0, 0.0, 1.0)
            Draw = (obj.x, obj.y, obj.z)
            CenterPoint = (obj.ox, obj.oy)
            Rotation = (obj.rx, obj.ry, obj.rz)


- obj 表 - 函式 -

        ○ obj.getobject(tag [, frame])
                取得具有指定標籤的其他物件的繪製資訊。
                ※ 標籤與目標項目「備註」中輸入的字串相符。
                ※ 指定第二個參數 frame 時，取得該時間軸影格的數值。省略時取得目前影格。
                ※ 當多個項目擁有相同標籤時，優先回傳在目標影格顯示中的項目。
                  若沒有顯示中的項目，則回傳第一個相符的項目。
                ※ 取得的值是項目自身的座標、旋轉與縮放。
                  不包含群組控制、攝影機控制以及上層特效造成的變換。
                ※ 各數值使用與 obj 表相同的單位體系。
                tag     : 設定於目標項目備註中的字串
                frame   : 要取得的時間軸絕對影格 (可省略)
                回傳值  : 物件資訊表。當不存在相符項目時回傳 nil。
                          .exist : 是否在目標影格顯示中 (boolean)
                          .x     : 顯示基準座標 X
                          .y     : 顯示基準座標 Y
                          .z     : 顯示基準座標 Z
                          .zoom  : 縮放率 (1.0 為原始大小)
                          .sx    : X 方向縮放率 (= zoom)
                          .sy    : Y 方向縮放率 (= zoom)
                          .rx    : X 軸旋轉角度 (度, 恆為 0)
                          .ry    : Y 軸旋轉角度 (度, 恆為 0)
                          .rz    : Z 軸旋轉角度 (度)
                          .rxr   : X 軸旋轉角度 (弧度, 恆為 0)
                          .ryr   : Y 軸旋轉角度 (弧度, 恆為 0)
                          .rzr   : Z 軸旋轉角度 (弧度)
                          .alpha : 不透明度 (0.0 ～ 255.0, 已套用淡入淡出)
                          .layer : 所在圖層編號
                範例：
                    local funnel = obj.getobject("funnel")
                    if funnel then
                        obj.x = funnel.x
                        obj.y = funnel.y
                    end
                固定為發射時座標的範例 (發射來源備註為 "funnel")：
                    local fire = timelineframe - frame
                    local src = obj.getobject("funnel", fire)
                    if src then
                        obj.x = src.x
                        obj.y = src.y
                    end

        ○ obj.getpixel(x, y)
                取得指定座標的像素資訊。
                * 首次呼叫時發生 GPU→CPU 的影像資料傳輸（存在處理負擔）。
                * 像素座標以影像左上角為 (0, 0)，向右和向下為正方向。
                * 傳回值為將預乘 Alpha 逆轉換後的直通 Alpha 等效值。
                x      ：X 座標（0 ～ obj.w - 1）
                y      ：Y 座標（0 ～ obj.h - 1）
                傳回值 ：r, g, b, a 共 4 個值（多值傳回）
                          r = 紅色分量（0.0 ～ 255.0）
                          g = 綠色分量（0.0 ～ 255.0）
                          b = 藍色分量（0.0 ～ 255.0）
                          a = Alpha 值（0.0 ～ 255.0）
                          * 超出範圍的座標傳回 (0, 0, 0, 0)。
                範例：r, g, b, a = obj.getpixel(10, 20)

        ○ obj.setpixel(x, y, r, g, b [, a])
                覆寫指定座標的像素資訊。
                * 呼叫時像素資料自動標記為「已修改」，
                  腳本結束後輸出修改後的影像。
                * 內部以預乘 Alpha 格式（BGRA）保存，
                  setpixel 時進行預乘轉換。
                x      ：X 座標（0 ～ obj.w - 1）
                y      ：Y 座標（0 ～ obj.h - 1）
                r      ：紅色分量（0.0 ～ 255.0）
                g      ：綠色分量（0.0 ～ 255.0）
                b      ：藍色分量（0.0 ～ 255.0）
                a      ：Alpha 值（省略時為 255.0）
                範例：obj.setpixel(10, 20, 255, 0, 0, 128)

        ○ obj.getpixeldata()
                傳回用於批次存取所有像素資料的代理物件。
                * 與 getpixel/setpixel 相比，像素操作速度更快。
                * 首次呼叫時發生 GPU→CPU 的影像資料傳輸（存在處理負擔）。
                傳回值 ：PixelDataProxy 物件
                          .width  ：影像水平像素數（= obj.w）
                          .height ：影像垂直像素數（= obj.h）
                          :get(index)        ：透過索引取得通道值
                          :set(index, value) ：透過索引設定通道值

                索引規則：
                    索引從 1 開始（遵循 Lua 慣例）。
                    通道順序為 R、G、B、A 循環。
                    pixel(x, y) 的 R 通道索引 =
                        (y * obj.w + x) * 4 + 1
                    pixel(x, y) 的 G 通道索引 =
                        (y * obj.w + x) * 4 + 2
                    pixel(x, y) 的 B 通道索引 =
                        (y * obj.w + x) * 4 + 3
                    pixel(x, y) 的 A 通道索引 =
                        (y * obj.w + x) * 4 + 4

                    * 修改 Alpha（通道 4）時，RGB 也會按比例轉換。
                    * 修改 RGB 通道時，會進行預乘轉換。
                    * 取得值與 get(index) 相同，為直通 Alpha 等效值。

                範例：
                    local pd = obj.getpixeldata()
                    local idx = (10 * obj.w + 20) * 4 + 1  -- (x=20, y=10) 的 R 通道
                    local r = pd:get(idx)
                    pd:set(idx, 128)

        ○ obj.putpixeldata()
                為相容性而定義的空實作函式（no-op）。
                * AviUtl 腳本中 getpixeldata/putpixeldata 組合使用，
                  但本外掛中 setpixel/set 的修改會自動反映，
                  因此無需呼叫此函式。

        ○ obj.rand(st, ed [, seed [, frame]])
                回傳指定範圍 st ～ ed 內整數的確定性亂數。
                * seed 與 frame 相同時始終回傳相同的值。
                * 省略 seed 時以 0 處理，省略 frame 時使用目前影格 (obj.frame)。
                * 在傳統引擎 (MoonSharp) 與高速執行階段 (LuaJIT) 上回傳相同的值。
                  若需要不依賴引擎的小數亂數，請使用 anim.rand。
                st      : 範圍下限（小數無條件捨去為整數）
                ed      : 範圍上限（小數無條件捨去為整數）
                seed    : 亂數種子（可省略，預設 0）
                frame   : 用於變化的影格（可省略，預設 obj.frame）
                回傳值  : st ～ ed 的整數
                範例：obj.ox = obj.ox + obj.rand(-10, 10, 1, obj.frame)

        ○ obj.load(目標, ...)
                將物件緩衝區替換為產生的內容。支援 "figure"(圖形) / "text"(文字) /
                "image"(影像檔) / "movie"(影片檔)。載入後 obj.w / obj.h 等會更新為產生
                尺寸，緊接著的 getpixel 亦可讀取。傳統引擎與高速引擎皆可執行。

                obj.load("figure", 名稱, 顏色, 大小 [, 線寬 [, 長寬比]])
                        產生圖形並替換緩衝區(置中)。
                        名稱   : 円 / 四角形 / 三角形 / 五角形 / 六角形 / 星形
                                 (亦可用英文 circle, rect, triangle, pentagon,
                                 hexagon, star)。
                        顏色   : 填滿色 0xRRGGBB。
                        大小   : 基準大小(像素)。
                        線寬   : 輪廓寬度(可省略)。0 為填滿。
                        長寬比 : 變形 -1.0 ～ 1.0(可省略)。
                        範例：obj.load("figure", "円", 0xff3366, 200)

                obj.load("text", 字串)
                        以 obj.setfont 指定的字型、大小、顏色繪製字串並替換緩衝區
                        (使用 DirectWrite 繪製)。
                        範例：obj.setfont("Yu Gothic UI", 48); obj.load("text", "字幕")

                obj.load("image", 路徑)
                        載入影像檔並替換緩衝區。
                        ※ 為安全起見會驗證檔頭，僅載入真正的影像
                          (PNG / JPEG / GIF / BMP / TIFF / WebP)。
                        範例：obj.load("image", "C:\\img\\logo.png")

                obj.load("movie", 路徑 [, 時間])
                        載入影片檔指定時間的影格並替換緩衝區
                        (使用 Media Foundation 解碼)。
                        時間 : 秒。省略時為 obj.time。
                        ※ 為安全起見會驗證檔頭，僅載入真正的影片
                          (MP4 / MOV / MKV / WebM / AVI / FLV / ASF)。
                        範例：obj.load("movie", "C:\\mov\\clip.mp4", obj.time)

        ○ obj.setfont(字型 [, 大小 [, 樣式 [, 顏色]]])
                設定 obj.load("text") 使用的字型。
                樣式 : 位元旗標。1=粗體, 2=斜體, 3=粗斜體。
                顏色 : 0xRRGGBB。
                範例：obj.setfont("Meiryo", 64, 1, 0xffffff)

        ○ obj.draw([ox, oy, oz [, 縮放 [, 不透明度 [, 長寬比]]]])
                將目前物件影像合成到累積緩衝區並繪製。
                多次呼叫會疊加繪製(殘影、粒子等)。
                ox, oy, oz : 相對中心的偏移(預設 0)。
                縮放       : 1.0 為原始大小(預設 1.0)。
                不透明度   : 0.0 ～ 1.0(預設 1.0)。
                長寬比     : -1.0 ～ 1.0(預設 0)。
                ※ 呼叫後會抑制標準繪製，輸出合成結果。

        ○ obj.drawpoly(x0,y0,z0, x1,y1,z1, x2,y2,z2, x3,y3,z3
                        [, u0,v0, u1,v1, u2,v2, u3,v3 [, 不透明度]])
                將物件影像以材質方式繪製到 4 個頂點指定的四邊形。
                頂點順序為左上→右上→右下→左下。省略 UV 時使用整幅影像。
                ※ 省略 UV 而僅提供第 13 個引數時，將其作為不透明度(12 個座標 + 不透明度)。
                ※ 使用仿射變換對應(平行四邊形精確，非平行四邊形為近似)。

        ○ obj.copybuffer(目標, 來源)
                在緩衝區之間複製影像。依識別字首字元判定。
                  "obj"      : 物件影像緩衝區
                  "tmp"      : 暫存緩衝區
                  "cache:名稱" : 具名快取(跨影格保留)
                範例：obj.copybuffer("tmp", "obj")  -- 儲存
                      obj.copybuffer("obj", "tmp")  -- 還原
                ※ 由於 YMM4 的結構，無法取得 "frm"(影格緩衝區 / 背景)。

        ○ obj.getvalue(目標)
                取得 obj 的目前值(與 obj.x、obj.track0 等相同)。
                目標   : "x","y","z","ox","oy","rx","ry","rz","zoom","aspect",
                         "alpha","sx","sy","track0"～"track3","time","frame" 等。
                回傳值 : 數值(未知目標回傳 0)。

        ○ obj.setoption(名稱, 值) / obj.getoption(名稱)
                設定 / 取得選項。未支援的項目也會被接受，不會中斷腳本
                (getoption 回傳設定值，未設定為 0)。
                目前會反映到繪製的項目：
                  "antialias" : 0 為最近鄰，其它為線性內插(obj.draw / obj.drawpoly)。
                  "blend" : 合成模式編號(obj.draw / obj.drawpoly)。與 YMM4 屬性
                    面板中「合成模式」下拉選單相同的順序(從 0 開始)：
                    0=一般,1=Dissolve,2=比較（暗）,3=相乘,4=Color burn,5=陰影,
                    6=比較（亮）,7=畫面,8=Color dodge,9=Linear dodge (Add),
                    10=相加,11=覆蓋,12=Soft light,13=Hard light,14=Vivid light,
                    15=明暗,16=Pin light,17=Hard mix,18=差異,19=Exclution,
                    20=相減,21=Division,22=Hue,23=Saturation,24=色,25=亮度,
                    26=Lighter color,27=Behind,28=Darker color,29=Clear,
                    30=Clipping by background,31=XOR,32=Color inversion mask。
                    超出範圍或無效的值將視為 0(一般)。
                  "draw_state" : 以布林值決定此效果是否輸出工作影像。true 輸出
                    工作影像，false 則原樣傳遞輸入。可用於不使用 obj.draw 或
                    setpixel 也輸出工作影像，或反過來抑制輸出。兩個引擎都可用。
                  "drawtarget" : 切換 obj.draw / obj.drawpoly 的繪製目標。
                    "framebuffer"(預設)合成到影格緩衝。"tempbuffer" 則就地合成到
                    工作用 tmp 緩衝。obj.setoption("drawtarget", "tempbuffer", w, h)
                    中的 w, h 是初始化 tmp 緩衝的尺寸，省略則不初始化。畫到 tmp 的
                    結果可用 obj.copybuffer("obj", "tmp") 取回工作影像。合成到
                    tempbuffer 使用一般(來源覆蓋)，blend 套用於 framebuffer 的繪製。
                    兩個引擎都可用。在 MoonSharp 中，合成到 tempbuffer 在 GPU 上
                    執行，GPU 無法使用時會自動切換到 CPU 合成。由於捨入方式不同，
                    合成結果在引擎之間可能存在極小差異。

        ○ obj.pixeloption(名稱, 值)
                將像素操作的選項作為狀態保存。

        ○ obj.setanchor(名稱, 個數 [, 選項, ..])
                在預覽上顯示錨點，可用滑鼠拖曳調整座標。調整後的座標會隨專案保存。
                名稱 : 以字串指定接收座標的 Lua 變數名。變數排列為
                    pos={x0,y0,x1,y1,…}，指定 "xyz" 時為 {x0,y0,z0,x1,y1,z1,…}。
                    座標以物件中心為原點。
                個數 : 指定錨點數量。最多 32 個。
                選項 : 可用字串列舉。
                    "line" : 用線連接錨點。
                    "loop" : 用線連接錨點並閉合一圈。
                    "star" : 將每個錨點與物件中心用線連接。
                    "arm"  : 從物件中心依序連接到各錨點。
                    "xyz"  : 以 3D 座標處理錨點。預設是 2D。
                回傳設置的錨點數量。兩個引擎都可用。
                名稱指定 "track" 的軌道條連動暫不支援，回傳 0。
                範例：
                  num = obj.track0
                  obj.setanchor("pos", num, "loop")
                  for i = 0, num - 1 do
                    obj.draw(pos[i * 2 + 1], pos[i * 2 + 2])
                  end

- AviUtl 相容的全域函式 -

        以下函式可在全域範圍中直接呼叫。
        在所有執行引擎（MoonSharp / 原生）上回傳完全相同的結果。

        RGB(r, g, b)
        RGB(col)
        RGB(r1, g1, b1, r2, g2, b2)
                在顏色資訊（0x000000～0xffffff）與紅、綠、藍（各 0～255）之間相互
                轉換。3 個引數時合成為一個顏色資訊，1 個引數時分解為 r, g, b 三個值。
                6 個引數時，回傳隨物件時間經過（obj.time / obj.totaltime）從第一種
                顏色向第二種顏色線性變化的顏色資訊。
                例：col = RGB(64, 192, 255)          -- 0x40C0FF
                例：r, g, b = RGB(0x0080FF)          -- 0, 128, 255
                例：col = RGB(64,192,255, 128,64,0)  -- 起始色→結束色

        HSV(h, s, v)
        HSV(col)
        HSV(h1, s1, v1, h2, s2, v2)
                在顏色資訊（0x000000～0xffffff）與色相（0～360）、飽和度（0～100）、
                明度（0～100）之間相互轉換。引數含義與 RGB() 相同。
                例：col = HSV(60, 100, 100)          -- 0xFFFF00
                例：h, s, v = HSV(0x0080FF)          -- 210, 100, 100

        OR(a, b) / AND(a, b) / XOR(a, b)
                以 32bit 整數進行位元運算（或、與、互斥或）。
                小數轉換為整數時，恰好 0.5 及以下會被捨去（2.50→2、2.51→3）。
                例：result = OR(8, 3)                -- 11

        SHIFT(a, shift)
                進行算術移位。shift 為正數時左移，為負數時右移。
                例：result = SHIFT(5, 3)             -- 40
                例：result = SHIFT(91, -2)           -- 22

        debug_print(text)
                將指定字串傳送到 OutputDebugString()。用於除錯。
                可透過 DebugView 等除錯監視工具查看訊息。
                此外，腳本發生執行錯誤時會自動傳送錯誤訊息。
                例：debug_print(string.format("x = %d", obj.x))

- 腳本範例 -

        ○ 淡入（預設腳本）
                obj.alpha = math.min(time * 255, 255)

        ○ 淡出
                obj.alpha = math.max((obj.totalframe - obj.frame) / obj.framerate * 255, 0)

        ○ 用 Track0 控制不透明度
                obj.alpha = obj.track0

        ○ 隨時間旋轉
                obj.rz = time * 90  -- 每秒旋轉 90 度

        ○ 反轉像素紅色分量
                for y = 0, obj.h - 1 do
                    for x = 0, obj.w - 1 do
                        local r, g, b, a = obj.getpixel(x, y)
                        obj.setpixel(x, y, 255 - r, g, b, a)
                    end
                end

        ○ 使用 getpixeldata 進行灰階轉換（高速）
                local pd = obj.getpixeldata()
                local w = pd.width
                local h = pd.height
                for y = 0, h - 1 do
                    for x = 0, w - 1 do
                        local base = (y * w + x) * 4
                        local r = pd:get(base + 1)
                        local g = pd:get(base + 2)
                        local b = pd:get(base + 3)
                        local a = pd:get(base + 4)
                        local gray = r * 0.299 + g * 0.587 + b * 0.114
                        pd:set(base + 1, gray)
                        pd:set(base + 2, gray)
                        pd:set(base + 3, gray)
                    end
                end

        ○ 用 Track0 控制縮放（1.0 = 等倍）
                obj.zoom = obj.zoom * (1 + obj.track0 / 100)


- AviUtl 腳本的部分相容 -

        提供相容層，使 AviUtl 動畫效果腳本(*.anm 系列)中涉及座標、縮放、旋轉、
        不透明度的變形以及亂數的部分，無需額外修改即可執行。當腳本中不包含
        AviUtl 專有寫法時，此相容層完全不介入(傳統腳本的行為不變)。

        ○ 腳本標頭指令的解釋
                解釋註解形式的參數宣告，在執行前定義腳本主體所參考的變數。
                --dialog:項目名,local 變數=預設值;…
                        將各項目的宣告部分(如 local 變數=預設值)展開至主體開頭。
                        在 AviUtl 中這些是透過對話框編輯的值，此處以預設值執行。
                --color:0xRRGGBB
                        以指定顏色定義變數 color。
                --check0:項目名,預設值
                        以預設值定義 obj.check0 (0～3)。
                --param:任意 Lua 陳述式
                        將所寫的 Lua 陳述式原樣展開至主體開頭。
                --track0: ～ --track3:
                        軌道列的值仍透過 obj.track0 ～ obj.track3 參考。標頭的標籤、
                        範圍與預設值不會反映至 UI。

        ○ 透過 @ 的多區段
                當一個腳本以 @名稱 定義了多個效果時，僅執行第一個區段。

        ○ obj.rand(st, ed [, seed [, frame]])
                與 AviUtl 相容的確定性整數亂數。詳情請參閱「obj 表 - 函式」。

        ○ 繪製·合成·載入
                支援 obj.draw / obj.drawpoly 的繪製合成、obj.copybuffer 的緩衝區操作、
                obj.load("figure"/"text"/"image"/"movie") 的產生與載入、obj.setfont、
                obj.effect、obj.getvalue 以及 obj.setoption / obj.getoption /
                obj.pixeloption。詳情請參閱「obj 表 - 函式」。

        ○ obj.effect 的 AviUtl / AviUtl2 相容對應表
                obj.effect("效果名", "參數名", 值, ...) 會先查閱相容對應表。
                以 AviUtl / AviUtl2 名稱宣告的效果，數值會經過宣告的倍率/位移轉換後再寫入
                YMM4 的屬性。對應表中未宣告的名稱則依 YMM4 的效果名或關鍵字直接解析，
                因此既有呼叫仍可正常使用。此對應表是依 YMM4 自身的 AviUtl 匯出
                (.exo 輸出) 的逆向建構的，因此往返一致。目標引擎透過 --!aviutl(預設) /
                --!aviutl2 切換。
                對應表中宣告的效果名及其主要參數如下。
                  ぼかし: 範囲, サイズ固定
                  境界ぼかし: 範囲
                  方向ブラー: 範囲, 角度, サイズ固定
                  放射ブラー: 範囲, X, Y, サイズを固定
                  閃光: 範囲, X, Y, サイズを固定
                  クロマキー: 色相範囲, 境界補正
                  クリッピング: 上, 下, 左, 右, 中心の位置を変更
                  斜めクリッピング: 中心X, 中心Y, 角度, ぼかし, 幅
                  領域拡張: 上, 下, 左, 右
                  座標: X, Y, Z
                  回転: X, Y, Z
                  リサイズ: 拡大率, X, Y, 補間なし
                  画像ループ: 横回数, 縦回数
                  極座標変換: 中心幅, 渦巻
                  グラデーション: 強さ, 中心X, 中心Y, 角度, 幅
                  単色化: 強さ, 輝度を保持する
                  縁取り: サイズ, ぼかし
                  反転: -
                  モザイク: サイズ
                  ディスプレイスメントマップ: ぼかし
                  動画ファイル合成: X, Y, 拡大率
                  色調補正: 明るさ, ｺﾝﾄﾗｽﾄ, 色相, 輝度, 彩度
                  色ずれ: ずれ幅, 角度, 強さ
                色調補正 與 色ずれ 對應社群外掛的效果。

        以下功能不在此相容層的範圍內。
        呼叫它們的腳本不會如預期運作。
                × 透過 require 載入模組，以及 io / dofile 等任意檔案存取
                  (obj.load 的影像 / 影片載入已附檔頭驗證支援)。
                × obj.setanchor 等。


- 不支援項目·限制事項 -

        × 不支援 AviUtl2 的 obj.rand1()、obj.mes()、obj.setanchor() 等。
        × 不支援腳本進行任意檔案存取（require、dofile、io 等）。
          僅 obj.load("image"/"movie") 會載入經檔頭驗證確認為真正媒體的檔案。
        × 修改 obj.z 會反映至繪製中（包含在 ReadBack 中），
          但 3D 繪製支援有限。
        × 超過腳本逾時時間（5 秒）時，執行將被強制終止。
          實作時需注意避免出現無限迴圈。
        × AviUtl2 的標頭指令僅部分支援。--dialog / --color / --check / --param 的
          預設值會被解釋（參閱「AviUtl 腳本的部分相容」），但 --track@ 的標籤、範圍
          與預設值不會反映至 UI，值透過 Track0 ～ Track3 使用。
